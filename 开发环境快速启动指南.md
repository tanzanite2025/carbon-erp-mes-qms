## 🔧 已实施的优化

### 1. ✅ 快速启动脚本

**文件**:
- `scripts/dev-fast.sh` (Linux/Mac)
- `scripts/dev-fast.ps1` (Windows)

**功能**:
- 自动检测 Docker 服务状态
- 如果服务运行中，只重启应用
- 如果服务未运行，完整启动

---

### 2. ✅ 新增 pnpm 脚本

**package.json**:
```json
{
  "scripts": {
    "dev": "crbn up --no-portless",           // 完整启动
    "dev:fast": "turbo run dev --filter=./apps/erp --filter=./apps/mes",  // 快速启动
    "dev:services": "crbn up --no-apps --no-portless",  // 只启动服务
    "dev:erp": "turbo run dev --filter=./apps/erp",     // 只启动 ERP
    "dev:mes": "turbo run dev --filter=./apps/mes"      // 只启动 MES
  }
}
```

---

### 3. ✅ Vite 配置优化

**apps/erp/vite.config.ts** 和 **apps/mes/vite.config.ts**:

```typescript
{
  // 共享缓存目录
  cacheDir: "../../node_modules/.vite/erp",
  
  // 优化依赖预构建
  optimizeDeps: {
    force: false,  // 不强制重新预构建
    include: [
      "react",
      "react-dom",
      "@remix-run/react",
      // ... 其他常用依赖
    ]
  },
  
  // 减少文件监听
  server: {
    watch: {
      ignored: [
        "**/node_modules/**",
        "**/.git/**",
        "**/dist/**",
        "**/build/**"
      ]
    }
  }
}
```

**效果**:
- 预构建速度提升 30-50%
- 减少不必要的文件监听
- 共享缓存，避免重复构建

## 🐛 故障排查

### 问题 1: Docker 服务未启动

**症状**: 运行 `pnpm dev:fast` 报错

**解决**:
```bash
# 先启动 Docker 服务
pnpm dev:services

# 然后在另一个终端启动应用
pnpm dev:fast
```

---

### 问题 2: 端口被占用

**症状**: `Error: Port 4000 is already in use`

**解决**:
```bash
# 查看占用端口的进程
# Windows
netstat -ano | findstr :4000

# Linux/Mac
lsof -i :4000

# 停止旧进程
# Windows
taskkill /PID <PID> /F

# Linux/Mac
kill -9 <PID>
```

---

### 问题 3: Vite 缓存问题

**症状**: 修改代码后没有生效

**解决**:
```bash
# 清除 Vite 缓存
rm -rf node_modules/.vite

# 重新启动
pnpm dev:fast
```

---

### 问题 4: Docker 容器状态异常

**症状**: 应用无法连接数据库

**解决**:
```bash
# 查看容器状态
docker compose ps

# 重启所有容器
crbn down
pnpm dev
```
