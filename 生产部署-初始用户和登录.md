# 生产部署：初始用户和登录流程

## 问题

部署到生产服务器后，如何创建第一个超级管理员用户并登录系统？

## Carbon 的用户管理机制

### 重要概念

Carbon **没有内置的超级管理员账号**。它使用以下机制：

1. **自助注册** - 用户通过邮箱自行注册（Cloud 版本）
2. **邀请制** - 管理员邀请员工加入（Enterprise 版本）
3. **OAuth 登录** - 通过 Google/Azure 登录

### 三种版本的区别

| 版本 | 用户注册方式 | 适用场景 |
|------|------------|---------|
| **Community** | 邮箱自助注册 | 开源社区版，任何人都可以注册 |
| **Cloud** | 邮箱自助注册 + 付费订阅 | SaaS 云服务 |
| **Enterprise** | 仅邀请制，不允许自助注册 | 私有部署 |

## 生产部署后的初始登录流程

### 方案 1：使用邮箱自助注册（推荐用于测试/开发）

#### 步骤

1. **访问登录页面**
   ```
   https://your-domain.com/login
   ```

2. **输入邮箱地址**
   - 输入你的邮箱（例如：`admin@yourcompany.com`）
   - 点击 "Sign in with Email"

3. **接收验证码**
   - 系统会发送验证码到你的邮箱
   - 如果是新用户，会自动创建账号

4. **完成 Onboarding**
   - 设置个人信息（姓名）
   - 选择界面主题
   - 创建公司信息
   - 选择订阅计划（Cloud 版本）

5. **成为第一个用户**
   - 第一个注册的用户自动成为公司管理员
   - 拥有所有权限

### 方案 2：使用 OAuth 登录（推荐用于生产）

#### 配置 Google OAuth

1. **在 Google Cloud Console 创建 OAuth 应用**
   - 访问 https://console.cloud.google.com/
   - 创建 OAuth 2.0 客户端 ID
   - 设置回调 URL：`https://your-domain.com/callback`

2. **配置环境变量**
   ```bash
   # .env
   AUTH_PROVIDERS=google
   GOOGLE_CLIENT_ID=your-google-client-id
   GOOGLE_CLIENT_SECRET=your-google-client-secret
   ```

3. **登录**
   - 访问 `/login`
   - 点击 "Sign in with Google"
   - 使用 Google 账号登录
   - 完成 Onboarding 流程

#### 配置 Azure OAuth（适合企业）

```bash
# .env
AUTH_PROVIDERS=azure
AZURE_CLIENT_ID=your-azure-client-id
AZURE_CLIENT_SECRET=your-azure-client-secret
AZURE_TENANT_ID=your-azure-tenant-id
```

### 方案 3：开发环境绕过登录（仅用于开发）

这是你当前使用的方法，**不适合生产环境**：

```bash
# .env
DEV_BYPASS_EMAIL=admin@example.com
```

登录时输入 `admin@example.com`，系统会自动创建用户并登录，无需验证。

**⚠️ 警告：** 生产环境必须删除 `DEV_BYPASS_EMAIL` 配置！

## 生产环境推荐配置

### 1. 邮件服务配置（必需）

Carbon 使用 Supabase Auth 发送邮件，需要配置邮件服务：

#### 选项 A：使用 Supabase 内置邮件（简单但有限制）

Supabase 提供免费的邮件服务，但有发送限制。

#### 选项 B：配置自定义 SMTP（推荐）

在 Supabase Dashboard 配置：

1. 访问 Supabase Dashboard
2. 进入 Authentication > Email Templates
3. 配置 SMTP 设置：
   ```
   SMTP Host: smtp.yourprovider.com
   SMTP Port: 587
   SMTP User: your-email@yourcompany.com
   SMTP Password: your-password
   ```

#### 选项 C：使用阿里云邮件推送（适合中国）

```bash
# 在 Supabase 中配置阿里云 SMTP
SMTP Host: smtpdm.aliyun.com
SMTP Port: 465
SMTP User: your-aliyun-email
SMTP Password: your-aliyun-password
```

### 2. 环境变量配置

```bash
# .env (生产环境)

# Supabase 配置
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE=your-service-role-key

# 应用 URL
ERP_URL=https://erp.yourcompany.com
MES_URL=https://mes.yourcompany.com

# 认证提供商（选择一个或多个）
AUTH_PROVIDERS=google,azure,passkey

# Google OAuth（可选）
GOOGLE_CLIENT_ID=your-google-client-id
GOOGLE_CLIENT_SECRET=your-google-client-secret

# Azure OAuth（可选）
AZURE_CLIENT_ID=your-azure-client-id
AZURE_CLIENT_SECRET=your-azure-client-secret
AZURE_TENANT_ID=your-azure-tenant-id

# 版本（根据你的许可证）
CARBON_EDITION=enterprise  # 或 cloud, community

# 默认语言
DEFAULT_LANGUAGE=zh

# ⚠️ 删除开发环境配置
# DEV_BYPASS_EMAIL=admin@example.com  # 生产环境必须删除！
```

### 3. 数据库初始化

Carbon 使用 Supabase 作为数据库，首次部署时会自动创建表结构。

#### 检查数据库是否初始化

```sql
-- 在 Supabase SQL Editor 中运行
SELECT * FROM public.user LIMIT 1;
SELECT * FROM public.company LIMIT 1;
```

如果表不存在，需要运行迁移：

```bash
# 在项目根目录
cd carbon
pnpm supabase:push
```

## 首次登录后的操作

### 1. 完成 Onboarding

第一个用户登录后会进入 Onboarding 流程：

1. **用户信息** (`/onboarding/user`)
   - 输入姓名
   - 设置个人资料

2. **界面主题** (`/onboarding/theme`)
   - 选择颜色主题（Zinc, Slate, Stone 等）
   - 选择明暗模式

3. **公司信息** (`/onboarding/company`)
   - 输入公司名称
   - 输入公司地址
   - 选择基础货币（CNY 人民币）
   - 输入公司网站

4. **订阅计划** (`/onboarding/plan`) - 仅 Cloud 版本
   - 选择订阅计划
   - 输入支付信息

### 2. 添加其他用户

完成 Onboarding 后，管理员可以添加员工：

#### 方法 A：通过邀请链接

1. 进入 **Users > Employees**
2. 点击 "New Employee"
3. 填写员工信息：
   - 姓名
   - 邮箱
   - 职位
   - 部门
4. 系统会发送邀请邮件
5. 员工点击邮件中的链接即可加入

#### 方法 B：直接创建用户（需要数据库访问）

```sql
-- 在 Supabase SQL Editor 中
-- 1. 创建 Auth 用户
INSERT INTO auth.users (
  instance_id,
  id,
  aud,
  role,
  email,
  encrypted_password,
  email_confirmed_at,
  created_at,
  updated_at
) VALUES (
  '00000000-0000-0000-0000-000000000000',
  gen_random_uuid(),
  'authenticated',
  'authenticated',
  'employee@yourcompany.com',
  crypt('temporary-password', gen_salt('bf')),
  now(),
  now(),
  now()
);

-- 2. 创建 Employee 记录
-- 需要先获取 companyId 和 userId
```

**⚠️ 不推荐：** 直接操作数据库容易出错，建议使用邀请功能。

### 3. 配置权限和角色

Carbon 使用基于角色的权限系统：

| 角色 | 权限 | 适用人员 |
|------|------|---------|
| **Owner** | 所有权限，包括删除公司 | 公司创始人 |
| **Admin** | 管理用户、配置系统 | IT 管理员 |
| **Employee** | 使用系统功能 | 普通员工 |

#### 设置用户角色

1. 进入 **Users > Employees**
2. 点击员工名称
3. 在 "Role" 下拉框中选择角色
4. 保存

## 常见问题

### Q1: 忘记管理员密码怎么办？

Carbon 使用 Magic Link 登录，没有密码。如果无法接收邮件：

1. 检查邮件服务配置
2. 检查垃圾邮件文件夹
3. 使用 Supabase Dashboard 重置用户

### Q2: 如何禁用自助注册？

设置版本为 Enterprise：

```bash
# .env
CARBON_EDITION=enterprise
```

Enterprise 版本只允许通过邀请添加用户。

### Q3: 可以使用手机号登录吗？

可以，但需要配置：

1. 在 Supabase Dashboard 启用 Phone Auth
2. 配置短信服务商（Twilio 或阿里云）
3. 修改登录页面支持手机号

详见：`中国本地化-手机号登录方案.md`

### Q4: 如何备份用户数据？

```bash
# 使用 Supabase CLI 备份
supabase db dump -f backup.sql

# 或者在 Supabase Dashboard 中导出
```

### Q5: 如何迁移现有用户？

如果你有现有的用户系统，可以：

1. 导出用户数据（邮箱、姓名等）
2. 使用 Supabase API 批量创建用户
3. 发送邀请邮件让用户激活账号

## 安全建议

### 1. 生产环境必须配置

- ✅ 删除 `DEV_BYPASS_EMAIL`
- ✅ 启用 HTTPS
- ✅ 配置 CORS 白名单
- ✅ 启用 Rate Limiting
- ✅ 配置 Cloudflare Turnstile（防机器人）

### 2. Supabase 安全配置

- ✅ 启用 Row Level Security (RLS)
- ✅ 限制 Service Role Key 的使用
- ✅ 定期轮换 API Keys
- ✅ 启用审计日志

### 3. 邮件安全

- ✅ 使用 SPF/DKIM/DMARC 验证
- ✅ 配置邮件发送限制
- ✅ 监控异常登录

## 总结

Carbon 的用户管理流程：

1. **首次部署** → 第一个用户通过邮箱或 OAuth 注册
2. **完成 Onboarding** → 创建公司和基础配置
3. **添加员工** → 通过邀请链接添加其他用户
4. **分配权限** → 设置用户角色和权限

**关键点：**
- ❌ 没有内置的超级管理员账号
- ✅ 第一个注册的用户自动成为 Owner
- ✅ 生产环境必须配置邮件服务
- ✅ 推荐使用 OAuth 登录（Google/Azure）
- ✅ Enterprise 版本禁用自助注册，只能通过邀请
