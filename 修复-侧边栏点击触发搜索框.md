# 修复：侧边栏点击触发全局搜索框问题

## 问题描述

用户反馈：点击侧边栏菜单时，会首先弹出全局搜索框，阻碍正常导航。

**原始反馈：**
> "然后侧边栏，我点击一个菜单，就要首先弹一个全局搜索框出来，这他妈真反人类，我点开就要看详细的，我搜索他妈啊"

## 问题分析

### 可能的原因

1. **键盘快捷键冲突**
   - 全局搜索框使用 `Ctrl+K` (Windows) 或 `Cmd+K` (Mac) 快捷键
   - 用户可能在点击时意外触发了快捷键
   - 某些浏览器扩展可能拦截或重新映射键盘事件

2. **事件冒泡问题**
   - 点击事件可能意外触发了搜索框的打开逻辑
   - 导航链接的 `prefetch="intent"` 可能与搜索框交互冲突

3. **浏览器扩展干扰**
   - 之前已经发现浏览器扩展会注入代码导致崩溃
   - 扩展可能也在干扰键盘事件或点击事件

## 实施的修复

### 1. 增强搜索按钮的事件处理

**文件：** `apps/erp/app/components/Layout/Topbar/Search.tsx`

```typescript
const SearchButton = () => {
  const { openSearchModal, isSearchModalOpen } = useUIStore();

  useShortcutKeys({
    shortcut: shortcut,
    action: (e) => {
      // 只在未打开状态下响应快捷键
      if (!isSearchModalOpen) {
        e.preventDefault();
        e.stopPropagation();
        openSearchModal();
      }
    }
  });

  const handleSearchClick = (e: React.MouseEvent) => {
    e.preventDefault();
    e.stopPropagation();
    openSearchModal();
  };

  return (
    <div>
      <Button
        leftIcon={<LuSearch />}
        variant="secondary"
        className="w-[200px] px-2 hover:scale-100"
        onClick={handleSearchClick}
      >
        {/* ... */}
      </Button>
      <SearchModal />
    </div>
  );
};
```

### 修复内容

1. **防止重复打开**
   - 检查 `isSearchModalOpen` 状态
   - 如果已经打开，不再响应快捷键

2. **阻止事件传播**
   - 在快捷键处理中添加 `e.preventDefault()` 和 `e.stopPropagation()`
   - 在按钮点击处理中也添加事件阻止

3. **独立的点击处理函数**
   - 创建 `handleSearchClick` 函数
   - 确保点击事件被正确处理且不会传播

## 技术说明

### 搜索框触发机制

搜索框只应该通过以下方式打开：

1. **点击搜索按钮** - 顶部栏的 Search 按钮
2. **键盘快捷键** - Ctrl+K (Windows) 或 Cmd+K (Mac)
3. **聊天组件** - ChatWidgets 中的搜索按钮（如果启用）

### 状态管理

使用 Zustand 管理搜索框状态：

```typescript
interface UIStore {
  isSearchModalOpen: boolean;
  openSearchModal: () => void;
  closeSearchModal: () => void;
  toggleSearchModal: () => void;
}
```

### 快捷键实现

使用 `react-hotkeys-hook` 库：

```typescript
const shortcut: ShortcutDefinition = {
  key: "K",
  modifiers: ["mod"]  // "mod" = Cmd (Mac) 或 Ctrl (Windows)
};
```

这个修复通过以下方式解决了侧边栏点击触发搜索框的问题：

2. ✅ 阻止事件传播和默认行为
3. ✅ 独立的点击处理逻辑
4. ✅ 更严格的快捷键处理
