# 翻译和语法错误修复总结

## 问题 1: root.tsx 语法错误

### 错误描述
在 `apps/erp/app/root.tsx` 中出现语法错误，导致 `lingui:extract` 命令失败：
```
SyntaxError: Unexpected token, expected "," (138:19)
```

### 根本原因
在之前添加 `SUPER_ADMIN_EMAIL` 时，`env` 对象被意外重复定义，导致对象结构错误：
```typescript
env: {
  // ... 属性
},
  // ❌ 这里多了一组重复的属性，没有对象包裹
  JIRA_CLIENT_ID,
  MES_URL,
  // ...
},
```

### 解决方案
删除重复的属性，保持正确的对象结构：
```typescript
env: {
  AUTH_PROVIDERS,
  CARBON_API_URL,
  // ... 所有属性
  SUPER_ADMIN_EMAIL,
  VERCEL_ENV,
  VERCEL_URL,
  XERO_CLIENT_ID
},
linguiCatalog,
mode: getMode(request),
// ...
```

## 技术细节

### Lingui 工作流程
1. **提取（Extract）**：从源代码中提取所有需要翻译的文本
2. **翻译（Translate）**：在 .po 文件中添加各语言的翻译
3. **编译（Compile）**：将 .po 文件编译为运行时使用的格式

### Pre-commit Hook
项目使用 Husky 配置了 pre-commit hook，会自动：
- 运行 `lingui:extract` 检查翻译完整性
- 如果有缺失的翻译，提交会失败
- 确保所有提交的代码都有完整的多语言支持
