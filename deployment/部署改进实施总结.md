## 🔧 使用指南

### 1. 部署新配置

```bash
# 1. 停止现有服务
pm2 stop all
docker compose down

# 2. 使用新的 Docker Compose
cd deployment
docker compose -f docker-compose.prod.yml up -d

# 3. 验证环境变量
source ../.env
./scripts/check-env.sh

# 4. 启动应用
pm2 start pm2/ecosystem.config.js
pm2 save

# 5. 启动监控
cd monitoring
docker compose up -d
```

### 2. 配置 PgBouncer

```bash
# 1. 生成密码哈希
echo -n "your-passwordpostgres" | md5sum

# 2. 编辑用户列表
nano deployment/pgbouncer/userlist.txt
# 添加: "postgres" "md5<hash>"

# 3. 启动 PgBouncer（已包含在 docker-compose.prod.yml）

# 4. 测试连接
psql -h 127.0.0.1 -p 6432 -U postgres postgres
```

### 3. 配置监控告警

```bash
# 1. 编辑监控脚本
nano deployment/scripts/monitor.sh

# 2. 设置告警邮箱
ALERT_EMAIL="admin@yourcompany.com"

# 3. 设置 Webhook（可选）
ALERT_WEBHOOK="https://hooks.slack.com/..."

# 4. 添加定时任务
crontab -e
# 添加: */5 * * * * /root/carbon/deployment/scripts/monitor.sh
```

### 4. 使用蓝绿部署

```bash
# 执行蓝绿部署
./deployment/scripts/blue-green-deploy.sh

# 如果失败，会自动回滚
```

### 5. 手动回滚

```bash
# 回滚到上一个版本
./deployment/scripts/rollback.sh HEAD~1

# 回滚数据库（可选）
# 脚本会提示选择备份文件
```

## 🔒 安全加固

### 1. 修改默认密码

```bash
# PostgreSQL
ALTER USER postgres WITH PASSWORD 'new-strong-password';

# Redis
# 编辑 redis.conf
requirepass your-strong-password

# Grafana
# 首次登录后立即修改
```

### 2. 配置防火墙

```bash
# 只允许本地访问数据库端口
sudo ufw deny 5432
sudo ufw deny 6379
sudo ufw deny 6432

# 允许 HTTP/HTTPS
sudo ufw allow 80/tcp
sudo ufw allow 443/tcp
```

### 3. 启用 SSL

```bash
# PostgreSQL SSL
# 编辑 postgresql.conf
ssl = on
ssl_cert_file = '/path/to/server.crt'
ssl_key_file = '/path/to/server.key'

# Redis SSL
# 使用 stunnel 或 Redis 6+ 内置 SSL
```
