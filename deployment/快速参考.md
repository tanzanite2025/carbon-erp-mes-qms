## 🚀 常用命令

### 应用管理

```bash
# 启动应用
pm2 start deployment/pm2/ecosystem.config.js

# 停止应用
pm2 stop all

# 重启应用
pm2 restart all

# 查看状态
pm2 status

# 查看日志
pm2 logs

# 监控
pm2 monit
```

### Docker 管理

```bash
# 启动所有服务
docker compose -f deployment/docker-compose.prod.yml up -d

# 停止所有服务
docker compose -f deployment/docker-compose.prod.yml down

# 查看状态
docker compose -f deployment/docker-compose.prod.yml ps

# 查看日志
docker compose -f deployment/docker-compose.prod.yml logs -f

# 重启服务
docker compose -f deployment/docker-compose.prod.yml restart
```

### Nginx 管理

```bash
# 测试配置
sudo nginx -t

# 重载配置
sudo systemctl reload nginx

# 重启 Nginx
sudo systemctl restart nginx

# 查看状态
sudo systemctl status nginx

# 查看日志
sudo tail -f /var/log/nginx/error.log
```

### 数据库管理

```bash
# 连接数据库
docker exec -it carbon-postgres psql -U postgres

# 备份数据库
./deployment/scripts/backup-db.sh

# 恢复数据库
./deployment/scripts/restore-db.sh /path/to/backup.sql.gz

# 查看连接数
docker exec carbon-postgres psql -U postgres -c "SELECT count(*) FROM pg_stat_activity;"

# 查看慢查询
docker exec carbon-postgres psql -U postgres -c "SELECT * FROM pg_stat_statements ORDER BY mean_exec_time DESC LIMIT 10;"
```

### Redis 管理

```bash
# 连接 Redis
docker exec -it carbon-redis redis-cli

# 查看信息
docker exec carbon-redis redis-cli info

# 查看内存使用
docker exec carbon-redis redis-cli info memory

# 查看慢查询
docker exec carbon-redis redis-cli slowlog get 10

# 清空缓存（慎用）
docker exec carbon-redis redis-cli FLUSHALL
```

## 📊 性能优化

### 数据库优化

```sql
-- 查看表大小
SELECT
  schemaname,
  tablename,
  pg_size_pretty(pg_total_relation_size(schemaname||'.'||tablename)) AS size
FROM pg_tables
WHERE schemaname = 'public'
ORDER BY pg_total_relation_size(schemaname||'.'||tablename) DESC
LIMIT 10;

-- 查看索引使用情况
SELECT
  schemaname,
  tablename,
  indexname,
  idx_scan,
  idx_tup_read,
  idx_tup_fetch
FROM pg_stat_user_indexes
ORDER BY idx_scan DESC;

-- 清理和分析
VACUUM ANALYZE;

-- 重建索引
REINDEX DATABASE postgres;
```

### Redis 优化

```bash
# 查看内存碎片率
docker exec carbon-redis redis-cli info memory | grep mem_fragmentation_ratio

# 手动碎片整理
docker exec carbon-redis redis-cli MEMORY PURGE

# 查看键空间
docker exec carbon-redis redis-cli info keyspace

# 查看最大内存
docker exec carbon-redis redis-cli config get maxmemory
```

## 🐛 故障排查

### 应用无法启动

```bash
# 1. 查看 PM2 日志
pm2 logs --lines 100

# 2. 检查端口占用
netstat -tlnp | grep 4000
netstat -tlnp | grep 4001

# 3. 检查环境变量
pm2 env 0

# 4. 手动启动测试
cd apps/erp
PORT=4000 npm start
```

### 数据库连接失败

```bash
# 1. 检查 PostgreSQL 状态
docker exec carbon-postgres pg_isready -U postgres

# 2. 查看连接数
docker exec carbon-postgres psql -U postgres -c "SELECT count(*) FROM pg_stat_activity;"

# 3. 查看日志
docker compose logs postgres

# 4. 重启 PostgreSQL
docker compose restart postgres
```

### Nginx 502 错误

```bash
# 1. 检查应用状态
pm2 status

# 2. 检查端口监听
netstat -tlnp | grep 4000

# 3. 查看 Nginx 错误日志
sudo tail -f /var/log/nginx/error.log

# 4. 测试上游服务
curl http://localhost:4000/health
```

### 内存不足

```bash
# 1. 查看内存使用
free -h

# 2. 查看进程内存
ps aux --sort=-%mem | head -10

# 3. 清理缓存
sudo sync && sudo sysctl -w vm.drop_caches=3

# 4. 重启应用
pm2 restart all
```

### 磁盘空间不足

```bash
# 1. 查看磁盘使用
df -h

# 2. 查找大文件
du -h / | sort -rh | head -20

# 3. 清理日志
sudo journalctl --vacuum-time=7d
pm2 flush

# 4. 清理 Docker
docker system prune -a
```
