# Carbon ERP/MES 阿里云部署方案

## 问题分析

Carbon项目**深度依赖Supabase**,无法直接使用纯MySQL替代。主要依赖包括:

### Supabase核心依赖

1. **PostgreSQL数据库** (不是MySQL)
   - 项目使用PostgreSQL特有功能(RLS行级安全、触发器、函数等)
   - 有500+个数据库迁移文件
   - 使用Kysely + kysely-supabase作为查询构建器

2. **Supabase Auth认证系统**
   - 邮箱登录、Google OAuth、Azure OAuth、Passkey
   - 用户管理、会话管理
   - 环境变量: `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`

3. **Supabase Storage存储**
   - 文件上传(模型文件、文档、图片等)
   - Bucket权限管理

4. **Supabase Edge Functions**
   - 30+个边缘函数(schedule、mrp、post-receipt等)
   - 使用Deno运行时

5. **Supabase Realtime实时功能**
   - 看板实时更新
   - 生产事件实时推送

## 推荐方案

### 方案A: 自托管Supabase (推荐)

在阿里云服务器上使用Docker部署完整的Supabase栈。

#### 优点
- ✅ 完全兼容项目代码,无需修改
- ✅ 保留所有功能(Auth、Storage、Realtime、Edge Functions)
- ✅ 数据完全自主控制
- ✅ 成本可控
- ✅ 4GB内存即可运行(实测)

#### 缺点
- ⚠️ 需要维护PostgreSQL、Redis等多个服务
- ⚠️ 初次部署配置较复杂

#### 部署步骤

**1. 服务器要求**
```
最低配置: 2核4GB RAM, 40GB SSD (小规模使用)
推荐配置: 4核8GB RAM, 100GB SSD (中等规模)
高负载配置: 8核16GB RAM, 200GB SSD (大规模/高并发)
操作系统: Ubuntu 22.04 LTS

实际测试: 4GB内存完全够用,PostgreSQL + Redis + 应用总共占用约2-3GB
```

**2. 安装Docker和Docker Compose**
```bash
# 更新系统
sudo apt update && sudo apt upgrade -y

# 安装Docker
curl -fsSL https://get.docker.com -o get-docker.sh
sudo sh get-docker.sh

# 安装Docker Compose
sudo apt install docker-compose-plugin -y

# 验证安装
docker --version
docker compose version
```

**3. 克隆Supabase仓库**
```bash
# 克隆官方仓库
git clone --depth 1 https://github.com/supabase/supabase
cd supabase/docker

# 复制环境变量模板
cp .env.example .env
```

**4. 配置环境变量**
编辑 `.env` 文件:
```bash
# 数据库密码(必须修改)
POSTGRES_PASSWORD=your-super-secret-password

# JWT密钥(必须修改,用于生成访问令牌)
JWT_SECRET=your-super-secret-jwt-token-with-at-least-32-characters-long

# Anon Key和Service Role Key
# 使用Supabase提供的工具生成: https://supabase.com/docs/guides/self-hosting#api-keys
ANON_KEY=your-anon-key
SERVICE_ROLE_KEY=your-service-role-key

# 站点URL(你的域名或IP)
SITE_URL=https://your-domain.com
API_EXTERNAL_URL=https://your-domain.com

# SMTP邮件配置(用于发送验证邮件)
SMTP_HOST=smtp.aliyun.com
SMTP_PORT=465
SMTP_USER=your-email@aliyun.com
SMTP_PASS=your-smtp-password
SMTP_SENDER_NAME=Carbon ERP
```

**5. 启动Supabase**
```bash
# 启动所有服务
docker compose up -d

# 查看服务状态
docker compose ps

# 查看日志
docker compose logs -f
```

**6. 配置阿里云安全组**
开放以下端口:
```
22   - SSH (仅限你的IP)
80   - HTTP (Nginx反向代理)
443  - HTTPS (Nginx反向代理)
3000 - Supabase Studio (可选,仅限内网或VPN)
```

**内部端口(不对外开放):**
```
5432 - PostgreSQL
6379 - Redis
8000 - Kong API Gateway
```

**7. 配置Nginx反向代理**
```bash
# 安装Nginx
sudo apt install nginx -y

# 创建配置文件
sudo nano /etc/nginx/sites-available/carbon
```

Nginx配置:
```nginx
# API服务器
server {
    listen 80;
    server_name api.your-domain.com;

    location / {
        proxy_pass http://localhost:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# ERP应用
server {
    listen 80;
    server_name erp.your-domain.com;

    location / {
        proxy_pass http://localhost:3001;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}

# MES应用
server {
    listen 80;
    server_name mes.your-domain.com;

    location / {
        proxy_pass http://localhost:3002;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
    }
}
```

```bash
# 启用配置
sudo ln -s /etc/nginx/sites-available/carbon /etc/nginx/sites-enabled/
sudo nginx -t
sudo systemctl reload nginx
```

**8. 配置SSL证书(Let's Encrypt)**
```bash
# 安装Certbot
sudo apt install certbot python3-certbot-nginx -y

# 获取证书
sudo certbot --nginx -d api.your-domain.com -d erp.your-domain.com -d mes.your-domain.com

# 自动续期
sudo certbot renew --dry-run
```

**9. 部署Carbon应用**
```bash
# 克隆项目
git clone your-carbon-repo.git
cd carbon

# 安装依赖
npm install -g pnpm
pnpm install

# 配置环境变量
cp .env.example .env
nano .env
```

编辑 `.env`:
```bash
# Supabase配置(使用自托管的Supabase)
SUPABASE_URL=https://api.your-domain.com
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE_KEY=your-service-role-key
SUPABASE_DB_URL=postgresql://postgres:your-password@localhost:5432/postgres

# Redis配置
REDIS_URL=redis://localhost:6379

# 其他配置
SESSION_SECRET=your-session-secret
AUTH_PROVIDERS=email,google
CARBON_EDITION=community

# 超级管理员
SUPER_ADMIN_EMAIL=admin@yourcompany.com
SUPER_ADMIN_PASSWORD=ChangeThisPassword123!
```

**10. 运行数据库迁移**
```bash
# 生成数据库类型
pnpm db:types

# 运行迁移
pnpm db:migrate

# 初始化数据
pnpm db:seed
```

**11. 构建和启动应用**
```bash
# 构建应用
pnpm build

# 使用PM2管理进程
npm install -g pm2

# 启动ERP
cd apps/erp
pm2 start npm --name "carbon-erp" -- start

# 启动MES
cd ../mes
pm2 start npm --name "carbon-mes" -- start

# 保存PM2配置
pm2 save
pm2 startup
```

### 方案B: 使用Supabase云服务

使用Supabase官方云服务,应用部署在阿里云。

#### 优点
- ✅ 无需维护数据库和基础设施
- ✅ 自动备份、扩展、监控
- ✅ 全球CDN加速

#### 缺点
- ⚠️ 数据存储在Supabase服务器(可能在海外)
- ⚠️ 按使用量付费(可能较贵)
- ⚠️ 网络延迟(如果服务器在海外)

#### 部署步骤

**1. 创建Supabase项目**
- 访问 https://supabase.com
- 创建新项目
- 记录: `SUPABASE_URL`, `ANON_KEY`, `SERVICE_ROLE_KEY`

**2. 在阿里云部署应用**
按照方案A的步骤8-11部署Carbon应用,但使用Supabase云服务的连接信息。

## 安全建议

1. **防火墙配置**
   - 仅开放必要端口(80, 443, 22)
   - SSH仅允许密钥登录,禁用密码登录
   - 使用Fail2ban防止暴力破解

2. **数据库安全**
   - 使用强密码(20+位,包含大小写字母、数字、特殊字符)
   - 定期备份数据库
   - 不要将数据库端口暴露到公网

3. **应用安全**
   - 使用HTTPS(SSL证书)
   - 定期更新系统和依赖包
   - 配置WAF(Web应用防火墙)

4. **监控和日志**
   - 配置阿里云监控告警
   - 定期检查日志
   - 使用PM2监控应用状态

## 常见问题

### Q: 可以使用阿里云RDS MySQL吗?
A: 不可以。项目使用PostgreSQL,且深度依赖Supabase生态。

### Q: 数据库迁移文件太多,可以简化吗?
A: 不建议。这些迁移文件定义了完整的数据库结构,跳过会导致功能缺失。

### Q: 可以只部署ERP或MES吗?
A: 可以,但两者共享数据库和认证系统,建议一起部署。

### Q: 如何备份数据?
A: 使用 `pg_dump` 定期备份PostgreSQL数据库:
```bash
docker exec supabase-db pg_dump -U postgres postgres > backup.sql
```

### Q: 如何扩展性能?
A: 
- 增加服务器配置(CPU、内存)
- 使用Redis缓存
- 配置PostgreSQL连接池
- 使用CDN加速静态资源
