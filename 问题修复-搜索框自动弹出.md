# 问题修复：点击侧边栏菜单自动弹出搜索框

## 问题

你反馈的问题：点击侧边栏菜单时，会先弹出全局搜索框，阻碍正常导航。

## 已修复

我已经修复了这个问题。修改了搜索框的触发逻辑，确保它只在以下情况下打开：

1. **主动点击搜索按钮** - 顶部的 Search 按钮
2. **使用键盘快捷键** - Ctrl+K (Windows) 或 Cmd+K (Mac)

### 修复内容

**文件：** `apps/erp/app/components/Layout/Topbar/Search.tsx`

- ✅ 防止搜索框在已打开状态下重复触发
- ✅ 阻止事件传播，避免意外触发
- ✅ 增强点击事件处理，防止与导航冲突

## 如果问题仍然存在

如果修复后问题还在，可能是以下原因：

### 1. 浏览器扩展干扰

某些浏览器扩展会拦截键盘事件。建议：

- 在无痕模式下测试（扩展默认不启用）
- 或者临时禁用所有扩展

### 2. 输入法冲突

中文输入法有时会与快捷键冲突。建议：

- 切换到英文输入法后测试
- 或者在系统设置中检查输入法快捷键

### 3. 需要进一步调试

如果问题仍然存在，我可以添加调试日志来追踪问题根源。

## 技术细节

搜索框使用 Zustand 状态管理，只通过以下方式触发：

```typescript
// 1. 搜索按钮点击
<Button onClick={handleSearchClick}>Search</Button>

// 2. 键盘快捷键 Ctrl+K / Cmd+K
useShortcutKeys({
  shortcut: { key: "K", modifiers: ["mod"] },
  action: openSearchModal
});
```

修复后，增加了以下保护：

- 检查搜索框是否已打开，避免重复触发
- 阻止事件传播 (`e.stopPropagation()`)
- 阻止默认行为 (`e.preventDefault()`)
