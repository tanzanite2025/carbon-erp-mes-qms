# 🔐 Carbon 登录说明

## 问题

你看到的错误 "Invalid email/password combination" 是因为 Carbon 使用**无密码登录**（Magic Link），不是传统的用户名密码登录。

## 登录方式

### 方式 1: 开发绕过邮箱（最简单）⭐

已在 `.env` 文件中配置了开发绕过邮箱：

```bash
DEV_BYPASS_EMAIL="admin@example.com"
```

**使用方法**：
1. 访问 http://localhost:4000
2. 输入邮箱: `admin@example.com`
3. 点击 "Sign in with Email"
4. **自动登录**，无需密码和验证！

---

### 方式 2: Magic Link 登录

**首次注册**：
1. 访问 http://localhost:4000
2. 输入任意邮箱（如 `test@example.com`）
3. 点击 "Sign in with Email"
4. 打开 https://mail.dev 查看验证邮件
5. 点击邮件中的验证链接
6. 完成注册

**后续登录**：
1. 输入邮箱
2. 点击 "Sign in with Email"
3. 打开 https://mail.dev 查看登录链接
4. 点击链接登录

---

## 为什么没有密码？

Carbon 使用 **Magic Link** 无密码登录方式：

### 优点
- ✅ 更安全（没有密码泄露风险）
- ✅ 更方便（不需要记密码）
- ✅ 更现代（类似 Slack、Notion 等）

### 工作原理
1. 输入邮箱
2. 系统发送一个临时登录链接到邮箱
3. 点击链接即可登录

---

## 本地开发特殊配置

### 开发绕过邮箱

在 `.env` 文件中配置：

```bash
DEV_BYPASS_EMAIL="admin@localhost"
```

**作用**：
- 使用此邮箱可以**直接登录**
- 不需要验证邮件
- 不需要密码
- 仅用于开发环境

### 本地邮件服务

所有邮件都发送到本地的 **Inbucket**：

- **地址**: https://mail.dev
- **作用**: 查看验证邮件和登录链接
- **特点**: 不会发送到真实邮箱

---

## 其他登录方式

Carbon 支持多种登录方式，在 `.env` 中配置：

```bash
AUTH_PROVIDERS="email"  # 当前只启用邮箱登录
```

可选的登录方式：
- `email` - Magic Link 邮箱登录 ✅ 已启用
- `google` - Google OAuth 登录
- `azure` - Microsoft/Outlook 登录
- `passkey` - 生物识别登录（指纹、Face ID）

## 常见问题

### Q: 为什么输入密码后提示错误？

**A**: Carbon 不使用密码登录，使用 Magic Link。不需要输入密码，只需要输入邮箱。

### Q: 我没有收到验证邮件？

**A**: 邮件不会发送到真实邮箱，而是发送到本地的 Inbucket。访问 https://mail.dev 查看。

### Q: 可以使用密码登录吗？

**A**: 不可以。Carbon 设计为无密码登录。如果需要快速登录，使用开发绕过邮箱 `admin@localhost`。

### Q: 开发绕过邮箱安全吗？

**A**: 仅用于本地开发环境。生产环境不应该配置 `DEV_BYPASS_EMAIL`。

## 源码说明

登录逻辑在 `apps/erp/app/routes/_public+/login.tsx`：

```typescript
// 开发绕过邮箱检查
const devBypassEmail = process.env.DEV_BYPASS_EMAIL;
if (
  devBypassEmail &&
  email.toLowerCase() === devBypassEmail.toLowerCase() &&
  user.data?.active
) {
  const authSession = await signInWithBypassEmail(email);
  if (authSession) {
    // 直接登录，无需验证
    return redirect(path.to.authenticatedRoot);
  }
}
```
