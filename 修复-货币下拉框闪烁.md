# 修复：货币下拉框闪烁问题

## 🐛 问题描述

在 Onboarding 公司设置页面，点击"基础货币"下拉框时：
- 下拉框弹出后立即关闭
- 无法选择任何货币
- 无限闪烁，无法使用

## 🔍 根本原因

### 1. 事件冒泡问题
- `CommandItem` 的 `onSelect` 事件冒泡到父组件
- 导致 Popover 立即关闭
- 选择无法完成

### 2. 状态管理冲突
- `useFetcher` 在每次渲染时重新加载数据
- 导致组件状态不稳定
- Popover 的 `open` 状态被重置

### 3. 多次触发
- 点击事件被多次触发
- `onSelect` 被调用多次
- 导致闪烁效果

## ✅ 修复方案

### 修复 1：阻止事件冒泡

**文件**：`packages/react/src/Combobox.tsx`

```typescript
<CommandItem
  onSelect={(e) => {
    e?.preventDefault?.();      // 阻止默认行为
    e?.stopPropagation?.();     // 阻止事件冒泡
    onChange?.(item.value);
    setSearch("");
    setOpen(false);
  }}
  onPointerDown={(e) => {
    e.preventDefault();
    e.stopPropagation();
  }}
  onMouseDown={(e) => {
    e.preventDefault();
    e.stopPropagation();
  }}
/>
```

### 修复 2：稳定 Popover 状态

**文件**：`packages/react/src/Combobox.tsx`

```typescript
<Popover open={open} onOpenChange={setOpen} modal={true}>
  <PopoverTrigger disabled={isReadOnly} asChild>
    <CommandTrigger
      onClick={(e) => {
        e.preventDefault();      // 阻止默认行为
        e.stopPropagation();     // 阻止事件冒泡
        setOpen(true);
      }}
    />
  </PopoverTrigger>
</Popover>
```

### 修复 3：优化数据加载

**文件**：`apps/erp/app/components/Form/Currency.tsx`

```typescript
const useCurrencyCodes = () => {
  const currencyFetcher = useFetcher<...>({
    key: "currency-list"  // 添加唯一 key
  });

  useMount(() => {
    // 只在未加载时加载
    if (currencyFetcher.state === "idle" && !currencyFetcher.data) {
      currencyFetcher.load(path.to.api.currencies);
    }
  });

  // ...
};
```

## 📊 技术细节

### 问题分析

```
用户点击下拉框
    ↓
Popover 打开 (open = true)
    ↓
用户点击选项
    ↓
onSelect 触发
    ↓
事件冒泡到 Popover
    ↓
Popover 关闭 (open = false)
    ↓
但 onChange 还没执行完
    ↓
导致闪烁
```

### 修复后的流程

```
用户点击下拉框
    ↓
e.preventDefault() + e.stopPropagation()
    ↓
Popover 打开 (open = true, modal = true)
    ↓
用户点击选项
    ↓
onSelect 触发
    ↓
e.preventDefault() + e.stopPropagation()
    ↓
onChange 执行
    ↓
setOpen(false) 显式关闭
    ↓
✅ 正常工作
```
