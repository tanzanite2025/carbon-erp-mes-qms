# 快速参考：用户管理

## 🎯 核心原则

**普通员工不需要注册公司，只需接受邀请！**

---

## 👤 角色说明

| 角色 | 如何加入 | 权限 |
|-----|---------|------|
| **管理员** | 首个完成 Onboarding 的用户 | 完整权限 |
| **员工** | 管理员邀请 | 根据员工类型 |
| **客户** | 管理员邀请 | 查看订单、文档 |
| **供应商** | 管理员邀请 | 查看采购订单 |

---

## 📋 操作流程

### 1️⃣ 管理员创建公司

```
访问 http://localhost:4000
登录 admin@example.com
完成 Onboarding
✅ 成为管理员
```

### 2️⃣ 管理员邀请员工

```
进入 /x/users/employees
点击 "New Employee"
填写：
  - 邮箱: employee@example.com
  - 姓名: 张三
  - 员工类型: Manager
  - 地点: Headquarters
保存
✅ 邀请邮件已发送
```

### 3️⃣ 员工接受邀请

```
打开 https://mail.dev
找到邀请邮件
点击邀请链接
点击 "Join {公司名称}"
✅ 自动加入公司
```

### 4️⃣ 员工日常登录

```
访问 http://localhost:4000
输入邮箱
收到 Magic Link
点击链接登录
✅ 进入系统
```

## 🔐 权限管理

### 员工类型（推荐设置）

```
Administrator  → 所有权限
Manager        → 大部分模块的完整权限
Supervisor     → 特定部门的完整权限
Operator       → 有限的操作权限
Viewer         → 只读权限
```

### 修改权限

**方式 1：批量修改**
```
/x/settings/users/employee-types
编辑员工类型
调整模块权限
✅ 该类型所有员工自动更新
```

**方式 2：单独修改**
```
进入员工详情页
调整该员工的权限
✅ 只影响该员工
```

## 🛠️ 常用操作

### 重新发送邀请

```
员工列表 → 找到员工 → "Resend Invite"
```

### 撤销邀请

```
员工列表 → 找到员工 → "Revoke Invite"
```

### 停用员工

```
员工详情页 → "Deactivate"
```

## ❓ 常见问题

### Q: 员工可以自己注册公司吗？
**A**: 不可以。只能通过邀请加入。

### Q: 邀请链接会过期吗？
**A**: 不会自动过期，但可以撤销。

### Q: 一个用户可以属于多个公司吗？
**A**: 可以。通过不同邀请加入多个公司。

### Q: 如何跳过邮件验证？
**A**: 使用 `admin@example.com`（开发环境专用）
