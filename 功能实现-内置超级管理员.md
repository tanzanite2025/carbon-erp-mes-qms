# 功能实现：内置超级管理员账号

## 实现完成 ✅

我已经为你实现了**内置超级管理员账号**功能，解决了生产环境首次登录的问题。

## 核心功能

### 1. 邮箱+密码登录

- 输入超级管理员邮箱后，自动显示密码输入框
- 使用密码验证，无需邮件服务
- 首次登录自动创建账号

### 2. 跨环境支持

- ✅ 开发环境可用
- ✅ 测试环境可用
- ✅ 生产环境可用

### 3. 自动创建用户

- 首次登录时自动在 Supabase Auth 创建用户
- 用户信息：
  - Email: 配置的超级管理员邮箱
  - Password: 配置的超级管理员密码
  - First Name: "Super"
  - Last Name: "Admin"

### 4. 安全验证

- 密码必须完全匹配
- 支持 Rate Limiting
- 密码不暴露给前端（只暴露邮箱）

### 2. ERP 登录页面

**文件：** `apps/erp/app/routes/_public+/login.tsx`

**后端逻辑：**
```typescript
export async function action({ request }: ActionFunctionArgs) {
  // ...
  const { email, password } = validation.data;
  
  // 检查是否是超级管理员
  const superAdminEmail = process.env.SUPER_ADMIN_EMAIL;
  const superAdminPassword = process.env.SUPER_ADMIN_PASSWORD;
  
  if (email === superAdminEmail && superAdminPassword) {
    // 验证密码
    if (password !== superAdminPassword) {
      return error("Invalid credentials");
    }
    
    // 如果用户不存在，自动创建
    if (!user.data) {
      await carbonClient.auth.signUp({
        email: superAdminEmail,
        password: superAdminPassword,
        options: {
          data: {
            firstName: "Super",
            lastName: "Admin"
          }
        }
      });
    }
    
    // 使用密码登录
    const { data } = await carbonClient.auth.signInWithPassword({
      email: superAdminEmail,
      password: superAdminPassword
    });
    
    // 设置 Session
    const authSession = {
      accessToken: data.session.access_token,
      refreshToken: data.session.refresh_token,
      expiresIn: data.session.expires_in,
      expiresAt: data.session.expires_at,
      userId: data.user.id,
      companyId: "",
      console: false
    };
    
    const sessionCookie = await setAuthSession(request, { authSession });
    return redirect("/x", {
      headers: [["Set-Cookie", sessionCookie]]
    });
  }
  
  // 其他登录逻辑...
}
```

**前端逻辑：**
```typescript
export default function LoginRoute() {
  const [emailInput, setEmailInput] = useState("");
  const [showPassword, setShowPassword] = useState(false);
  
  // 检查是否是超级管理员邮箱
  const superAdminEmail = window.env?.SUPER_ADMIN_EMAIL;
  const isSuperAdmin = emailInput === superAdminEmail;
  
  useEffect(() => {
    setShowPassword(isSuperAdmin);
  }, [isSuperAdmin]);
  
  return (
    <ValidatedForm>
      <Input
        name="email"
        onChange={(e) => setEmailInput(e.target.value)}
      />
      
      {showPassword && (
        <Input
          name="password"
          type="password"
          placeholder="Password"
        />
      )}
      
      <Submit>Sign in</Submit>
    </ValidatedForm>
  );
}
```

## 工作流程

```
用户访问登录页面
    ↓
输入邮箱地址
    ↓
前端检测：是否是超级管理员邮箱？
    ├─ 是 → 显示密码输入框
    └─ 否 → 使用 Magic Link 登录
    ↓
用户输入密码
    ↓
提交表单
    ↓
后端验证：
    1. 检查邮箱是否匹配 SUPER_ADMIN_EMAIL
    2. 检查密码是否匹配 SUPER_ADMIN_PASSWORD
    3. 如果用户不存在，调用 Supabase Auth 创建用户
    4. 使用 signInWithPassword 登录
    5. 设置 Session Cookie
    ↓
重定向到 /x（或 Onboarding）
```

## 安全特性

### 1. 密码不暴露给前端

```typescript
// 只暴露邮箱，不暴露密码
export function getBrowserEnv() {
  return {
    SUPER_ADMIN_EMAIL,  // ✅ 暴露
    // SUPER_ADMIN_PASSWORD  // ❌ 不暴露
  };
}
```

### 2. 密码验证在后端

```typescript
// 后端验证密码
if (password !== superAdminPassword) {
  return error("Invalid credentials");
}
```

### 3. 支持 Rate Limiting

```typescript
const ratelimit = new Ratelimit({
  redis,
  limiter: Ratelimit.slidingWindow(RATE_LIMIT, "1 h"),
  analytics: true
});
```

### 4. 使用 Supabase Auth

- 密码存储在 Supabase Auth（加密）
- 使用标准的 `signInWithPassword` 方法
- Session 管理由 Supabase 处理
