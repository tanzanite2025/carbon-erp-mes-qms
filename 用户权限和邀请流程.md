# Carbon 用户权限和邀请流程详解

## 🎯 核心概念

Carbon 采用**基于邀请的多租户架构**，确保：
- ✅ 只有管理员可以创建公司
- ✅ 员工通过邀请链接加入公司
- ✅ 员工无需自己注册公司
- ✅ 完善的权限管理系统

## 👥 用户角色

### 1. 公司管理员（Company Admin）
- 第一个注册并完成 Onboarding 的用户
- 拥有完整的公司管理权限
- 可以邀请员工、客户、供应商

### 2. 员工（Employee）
- 通过管理员邀请加入
- 根据员工类型（Employee Type）分配权限
- 可以是：操作员、经理、主管等

### 3. 客户（Customer）
- 外部客户的联系人
- 有限的查看权限（订单、文档等）

### 4. 供应商（Supplier）
- 外部供应商的联系人
- 有限的查看权限（采购订单、文档等）

### 5. 控制台操作员（Console Operator）
- 仅用于 MES 终端的轻量级用户
- 无需邮箱和密码
- 可以在终端打卡和操作
- 后续可转换为正式员工

## 📋 完整的用户流程

### 场景 1：管理员首次设置

```
1. 访问 http://localhost:4000
2. 使用邮箱登录（开发环境：admin@example.com）
3. 完成 Onboarding：
   - 选择界面风格
   - 填写公司信息
   - 设置基础货币
4. 进入主界面
5. 成为公司管理员
```

### 场景 2：管理员邀请员工

```
管理员操作：
1. 进入 /x/users/employees
2. 点击 "New Employee"
3. 填写员工信息：
   - 邮箱
   - 姓名
   - 员工类型（Employee Type）
   - 工作地点
4. 点击保存

系统自动：
1. 创建用户账号（如果邮箱不存在）
2. 创建员工记录（状态：未激活）
3. 生成邀请码
4. 发送邀请邮件到员工邮箱
```

### 场景 3：员工接受邀请

```
员工操作：
1. 收到邀请邮件
2. 点击邮件中的邀请链接
   格式：http://localhost:4000/invite/{code}
3. 看到欢迎页面：
   "Welcome to Carbon"
   "Join {公司名称}"
4. 点击 "Join" 按钮

系统自动：
1. 验证邀请码
2. 激活员工账号
3. 关联用户到公司
4. 设置权限
5. 发送 Magic Link 登录邮件
6. 员工点击登录链接进入系统
```

### 场景 4：员工日常登录

```
1. 访问 http://localhost:4000
2. 输入邮箱
3. 收到 Magic Link 邮件
4. 点击链接登录
5. 自动进入所属公司
```

## 🔐 权限系统

### 权限结构

```typescript
{
  "users_view": ["company-id"],      // 查看用户
  "users_create": ["company-id"],    // 创建用户
  "users_update": ["company-id"],    // 更新用户
  "users_delete": ["company-id"],    // 删除用户
  "parts_view": ["company-id"],      // 查看零件
  "parts_create": ["company-id"],    // 创建零件
  // ... 其他模块权限
}
```

### 员工类型（Employee Type）

Carbon 使用**员工类型**来批量管理权限：

| 员工类型 | 典型权限 | 用途 |
|---------|---------|------|
| Administrator | 所有模块的完整权限 | 公司管理员 |
| Manager | 大部分模块的查看和编辑权限 | 部门经理 |
| Supervisor | 特定模块的查看和编辑权限 | 主管 |
| Operator | 有限的查看和操作权限 | 操作员 |
| Viewer | 只读权限 | 只读用户 |

### 权限继承

```
公司（Company）
  └─ 员工类型（Employee Type）
      └─ 模块权限（Module Permissions）
          ├─ View（查看）
          ├─ Create（创建）
          ├─ Update（更新）
          └─ Delete（删除）
```

## 🛠️ 管理员操作指南

### 1. 邀请新员工

**路径**：`/x/users/employees` → "New Employee"

**步骤**：
1. 填写员工信息
2. 选择员工类型（决定权限）
3. 选择工作地点
4. 保存

**结果**：
- 员工收到邀请邮件
- 员工状态：未激活（Inactive）
- 等待员工接受邀请

### 2. 重新发送邀请

**场景**：员工没收到邮件或邮件过期

**操作**：
1. 在员工列表中找到该员工
2. 点击 "Resend Invite"
3. 系统重新发送邀请邮件

### 3. 撤销邀请

**场景**：不再需要邀请该员工

**操作**：
1. 在员工列表中找到该员工
2. 点击 "Revoke Invite"
3. 邀请链接失效

### 4. 管理员工权限

**方式 1：通过员工类型**（推荐）
1. 进入 `/x/settings/users/employee-types`
2. 编辑员工类型
3. 调整模块权限
4. 该类型的所有员工自动继承新权限

**方式 2：单独调整**
1. 进入员工详情页
2. 调整该员工的权限
3. 只影响该员工

### 5. 创建控制台操作员

**场景**：MES 车间需要简单的打卡用户

**特点**：
- 无需邮箱
- 无需密码
- 只能在 MES 终端使用
- 后续可转换为正式员工

**操作**：
1. 进入 `/x/users/employees`
2. 选择 "Console Operator"
3. 填写姓名
4. 选择员工类型和地点
5. 保存

## 🗄️ 数据库结构

### 关键表

#### 1. `user` - 用户表
```sql
- id (UUID)
- email (唯一)
- firstName
- lastName
- avatarUrl
- active (是否激活)
- isConsoleOperator (是否为控制台操作员)
```

#### 2. `employee` - 员工表
```sql
- id (用户ID)
- companyId (公司ID)
- employeeTypeId (员工类型ID)
- active (是否激活)
```

#### 3. `invite` - 邀请表
```sql
- code (邀请码，UUID)
- email (被邀请人邮箱)
- companyId (公司ID)
- role (角色：employee/customer/supplier)
- permissions (权限JSON)
- createdBy (邀请人ID)
- acceptedAt (接受时间)
- revokedAt (撤销时间)
```

#### 4. `userToCompany` - 用户公司关联表
```sql
- userId (用户ID)
- companyId (公司ID)
- role (角色)
```

#### 5. `employeeType` - 员工类型表
```sql
- id (UUID)
- name (类型名称)
- companyId (公司ID)
```

#### 6. `employeeTypePermission` - 员工类型权限表
```sql
- employeeTypeId (员工类型ID)
- module (模块名称)
- view (查看权限)
- create (创建权限)
- update (更新权限)
- delete (删除权限)
```

## ❓ 常见问题

### Q1: 员工可以自己注册公司吗？

**A**: 不可以。只有第一个完成 Onboarding 的用户才能创建公司。其他用户必须通过邀请加入。

### Q2: 如果员工邮箱已经在系统中存在怎么办？

**A**: 系统会：
1. 检查该用户是否已经是该公司的员工
2. 如果是，提示"用户已经是员工"
3. 如果不是，将该用户添加到新公司（支持多公司）

### Q3: 邀请链接会过期吗？

**A**: 邀请链接不会自动过期，但可以被管理员撤销。

### Q4: 员工可以属于多个公司吗？

**A**: 可以。一个用户可以通过不同的邀请加入多个公司。

### Q5: 如何修改员工权限？

**A**: 两种方式：
1. 修改员工类型的权限（影响所有该类型的员工）
2. 单独修改某个员工的权限

### Q6: 控制台操作员和普通员工有什么区别？

**A**: 
- 控制台操作员：无邮箱，无密码，只能在 MES 终端使用
- 普通员工：有邮箱，可以登录 Web 界面

### Q7: 如何将控制台操作员转换为正式员工？

**A**: 系统提供 `convertConsoleOperatorToUser` 函数：
1. 为操作员分配真实邮箱
2. 创建 Supabase Auth 账号
3. 发送邀请邮件
4. 操作员可以登录 Web 界面

### Q8: 开发环境如何跳过邮件验证？

**A**: 使用 `DEV_BYPASS_EMAIL="admin@example.com"`：
- 该邮箱登录时自动创建用户
- 无需邮件验证
- 仅用于开发环境

## 🎯 最佳实践

### 1. 员工类型设计

**推荐结构**：
```
Administrator (管理员)
  └─ 所有权限

Manager (经理)
  └─ 大部分模块的完整权限
  └─ 无法管理用户和公司设置

Supervisor (主管)
  └─ 特定部门的完整权限
  └─ 如：生产主管、质量主管

Operator (操作员)
  └─ 有限的操作权限
  └─ 无法删除数据

Viewer (查看者)
  └─ 只读权限
  └─ 用于外部审计或临时访问
```

### 2. 邀请流程

1. **批量邀请**：一次邀请多个员工
2. **模板邮件**：自定义邀请邮件内容
3. **自动提醒**：未接受邀请的自动提醒
4. **权限预设**：为不同部门预设权限模板

### 3. 安全建议

1. **定期审查**：定期检查员工权限
2. **最小权限**：只给必要的权限
3. **离职处理**：及时停用离职员工账号
4. **审计日志**：记录权限变更历史
