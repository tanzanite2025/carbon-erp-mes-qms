# 中国本地化：手机号登录方案

## 🎯 问题

在中国，**邮箱不是主流的身份认证方式**。企业用户更习惯使用：

1. **手机号** - 最常用，人人都有
2. **企业微信/钉钉** - 企业内部通讯
3. **工号** - 企业内部编号

当前 Carbon 只支持邮箱登录，这在中国市场是一个严重的用户体验问题。

---

## ✅ 解决方案

### 方案对比

| 方案 | 优点 | 缺点 | 推荐度 |
|-----|------|------|--------|
| **手机号 + 短信验证码** | 用户习惯，无需邮箱 | 需要短信服务商，有成本 | ⭐⭐⭐⭐⭐ |
| **企业微信/钉钉登录** | 企业内部已有账号 | 需要企业认证，集成复杂 | ⭐⭐⭐⭐ |
| **工号 + 密码** | 简单，无外部依赖 | 安全性较低，需要管理密码 | ⭐⭐⭐ |
| **手机号 + 邮箱混合** | 兼容国内外 | 实现复杂 | ⭐⭐⭐⭐ |

### 实施步骤

#### 阶段 1：配置 Supabase Phone Auth

**1. 选择短信服务商**

**国内推荐**：
- **阿里云短信**：https://www.aliyun.com/product/sms
  - 价格：0.045元/条
  - 到达率高
  - 审核快

**国际备选**：
- **Twilio**：https://www.twilio.com/sms
  - 价格：$0.0079/条
  - Supabase 原生支持

**2. 配置 Supabase**

编辑 `packages/database/supabase/config.toml`：

```toml
[auth.sms]
# 启用手机号认证
enable_signup = true
enable_confirmations = true
max_frequency = "1m"  # 1分钟内只能发送一次

# 使用 Twilio（示例）
[auth.sms.twilio]
enabled = true
account_sid = "env(TWILIO_ACCOUNT_SID)"
message_service_sid = "env(TWILIO_MESSAGE_SERVICE_SID)"
auth_token = "env(TWILIO_AUTH_TOKEN)"
```

#### 阶段 2：修改登录界面

**1. 创建手机号登录组件**

`apps/erp/app/routes/_public+/login-phone.tsx`：

```typescript
import { PhoneInput } from "@carbon/form";
import { Button, Input, VStack } from "@carbon/react";
import { Trans, useLingui } from "@lingui/react/macro";
import { useState } from "react";
import { Form } from "react-router";

export default function PhoneLogin() {
  const { t } = useLingui();
  const [step, setStep] = useState<"phone" | "otp">("phone");
  const [phone, setPhone] = useState("");

  const sendOTP = async () => {
    // 调用 Supabase Phone Auth
    const { error } = await supabase.auth.signInWithOtp({
      phone: phone,
    });

    if (!error) {
      setStep("otp");
    }
  };

  const verifyOTP = async (otp: string) => {
    const { error } = await supabase.auth.verifyOtp({
      phone: phone,
      token: otp,
      type: "sms",
    });

    if (!error) {
      // 登录成功，重定向
      window.location.href = "/x";
    }
  };

  return (
    <VStack spacing={4}>
      {step === "phone" ? (
        <>
          <PhoneInput
            name="phone"
            label={t`手机号`}
            value={phone}
            onChange={setPhone}
            defaultCountry="CN"
          />
          <Button onClick={sendOTP}>
            <Trans>发送验证码</Trans>
          </Button>
        </>
      ) : (
        <>
          <Input
            name="otp"
            label={t`验证码`}
            placeholder={t`请输入6位验证码`}
            maxLength={6}
          />
          <Button onClick={() => verifyOTP(otp)}>
            <Trans>登录</Trans>
          </Button>
          <Button variant="ghost" onClick={() => setStep("phone")}>
            <Trans>重新发送</Trans>
          </Button>
        </>
      )}
    </VStack>
  );
}
```

## 🔧 阿里云短信集成（推荐）

### 为什么选择阿里云？

- ✅ 国内到达率最高
- ✅ 价格便宜（0.045元/条）
- ✅ 审核快速
- ✅ 稳定可靠

### 集成步骤

**1. 注册阿里云账号**

访问：https://www.aliyun.com/product/sms

**2. 创建短信签名和模板**

```
签名：【您的公司名】
模板：您的验证码是${code}，5分钟内有效。
```

**3. 创建 AccessKey**

在阿里云控制台创建 AccessKey ID 和 Secret

**4. 安装 SDK**

```bash
pnpm add @alicloud/dysmsapi20170525 @alicloud/openapi-client
```

**5. 创建短信服务**

`packages/lib/src/aliyun-sms.server.ts`：

```typescript
import Dysmsapi, * as $Dysmsapi from '@alicloud/dysmsapi20170525';
import * as $OpenApi from '@alicloud/openapi-client';

const config = new $OpenApi.Config({
  accessKeyId: process.env.ALIYUN_SMS_ACCESS_KEY_ID,
  accessKeySecret: process.env.ALIYUN_SMS_ACCESS_KEY_SECRET,
  endpoint: 'dysmsapi.aliyuncs.com',
});

const client = new Dysmsapi(config);

export async function sendSMS({
  phone,
  code,
}: {
  phone: string;
  code: string;
}) {
  const request = new $Dysmsapi.SendSmsRequest({
    phoneNumbers: phone,
    signName: process.env.ALIYUN_SMS_SIGN_NAME,
    templateCode: process.env.ALIYUN_SMS_TEMPLATE_CODE,
    templateParam: JSON.stringify({ code }),
  });

  try {
    const response = await client.sendSms(request);
    return { success: true, data: response };
  } catch (error) {
    console.error('发送短信失败:', error);
    return { success: false, error };
  }
}
```

**6. 集成到 Supabase Auth**

创建自定义 SMS Hook：

`packages/database/supabase/functions/send-sms/index.ts`：

```typescript
import { serve } from 'https://deno.land/std@0.168.0/http/server.ts';
import { sendSMS } from './aliyun-sms.ts';

serve(async (req) => {
  const { phone, token } = await req.json();
  
  const result = await sendSMS({
    phone,
    code: token,
  });

  return new Response(
    JSON.stringify(result),
    { headers: { 'Content-Type': 'application/json' } }
  );
});
```

配置 Supabase 使用自定义 Hook：

```toml
[auth.hook.send_sms]
enabled = true
uri = "http://localhost:54321/functions/v1/send-sms"
```

## 📊 成本估算

### 短信成本

| 场景 | 月用量 | 单价 | 月成本 |
|-----|--------|------|--------|
| 小型企业（50人） | 1000条 | ¥0.045 | ¥45 |
| 中型企业（200人） | 4000条 | ¥0.045 | ¥180 |
| 大型企业（1000人） | 20000条 | ¥0.045 | ¥900 |

### 优化建议

1. **验证码有效期**：5分钟（减少重复发送）
2. **发送频率限制**：1分钟内只能发送1次
3. **IP限制**：同一IP每天最多发送10次
4. **缓存验证码**：Redis 缓存，减少数据库查询
