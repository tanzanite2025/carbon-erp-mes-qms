## 📊 货币系统架构

### 1. 两层货币结构

```
┌─────────────────────────────────────────┐
│  currencyCode 表（货币代码参考表）        │
│  - 全局货币列表（USD, CNY, EUR等）       │
│  - 只读，所有公司共享                    │
│  - 包含：代码、名称、符号                │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  company 表                             │
│  - baseCurrencyCode: "USD"              │
│  - 每个公司的基础货币                    │
│  - 必填字段，不能为空                    │
└─────────────────────────────────────────┘
                  ↓
┌─────────────────────────────────────────┐
│  currency 表（公司启用的货币）            │
│  - 公司可以启用多个货币                  │
│  - 包含汇率信息                          │
│  - 可以动态添加/删除                     │
└─────────────────────────────────────────┘
```

### 2. 数据库结构

#### currencyCode 表（全局货币参考）
```sql
CREATE TABLE "currencyCode" (
    "code" TEXT PRIMARY KEY,      -- USD, CNY, EUR等
    "name" TEXT NOT NULL           -- US Dollar, Chinese Yuan等
);

-- 预置了120+种货币
INSERT INTO "currencyCode" VALUES
  ('USD', 'US Dollar'),
  ('CNY', 'Chinese Yuan'),
  ('EUR', 'Euro'),
  -- ... 更多货币
```

#### company 表（公司基础货币）
```sql
ALTER TABLE "company" 
ADD COLUMN "baseCurrencyCode" TEXT NOT NULL;

-- 外键约束
ALTER TABLE "company" 
ADD CONSTRAINT "company_baseCurrencyCode_fkey" 
  FOREIGN KEY ("baseCurrencyCode") 
  REFERENCES "currencyCode"("code");
```

#### currency 表（公司启用的货币）
```sql
CREATE TABLE "currency" (
  "id" TEXT PRIMARY KEY,
  "code" TEXT NOT NULL,           -- 货币代码
  "companyId" TEXT NOT NULL,      -- 所属公司
  "exchangeRate" NUMERIC(10,4),   -- 汇率
  "exchangeRateUpdatedAt" TIMESTAMP,
  
  FOREIGN KEY ("code") REFERENCES "currencyCode"("code"),
  FOREIGN KEY ("companyId") REFERENCES "company"("id")
);
```

## 🔍 为什么必须选择基础货币？

### 1. 财务核算基础

**基础货币是公司财务报表的核算货币**：

```
所有财务数据最终都要转换为基础货币：
- 资产负债表
- 损益表
- 现金流量表
- 成本核算
- 利润计算
```

### 2. 多货币交易的参照点

```
客户报价：EUR 1000
  ↓ 汇率转换
基础货币：USD 1100 (假设汇率1.1)
  ↓
财务报表：以 USD 显示
```

### 3. 数据一致性

```
场景：公司同时有 USD、EUR、CNY 交易

如果没有基础货币：
❌ 无法汇总总收入
❌ 无法计算总利润
❌ 无法生成财务报表

有基础货币：
✅ 所有交易转换为 USD
✅ 可以汇总计算
✅ 财务报表统一
```

## 💡 基础货币 vs 交易货币

### 基础货币（Base Currency）

- **定义**：公司财务核算的主货币
- **数量**：每个公司只有一个
- **用途**：
  - 财务报表
  - 成本核算
  - 利润计算
  - 内部报告

### 交易货币（Transaction Currency）

- **定义**：与客户/供应商交易时使用的货币
- **数量**：可以有多个
- **用途**：
  - 报价单
  - 销售订单
  - 采购订单
  - 发票

### 示例

```
公司：中国制造企业
基础货币：CNY（人民币）

交易货币：
- 国内客户：CNY
- 美国客户：USD
- 欧洲客户：EUR
- 日本客户：JPY

所有交易最终转换为 CNY 进行财务核算
```

## 🔄 多货币支持

### 1. 启用多个货币

**路径**：`/x/settings/accounting/currencies`

```
公司可以启用多个货币：
┌────────────────────────────────────┐
│ 启用的货币                          │
├────────┬──────────┬────────────────┤
│ USD    │ 1.0000   │ 基础货币        │
│ CNY    │ 7.2500   │ 2024-01-15     │
│ EUR    │ 0.9200   │ 2024-01-15     │
│ JPY    │ 149.50   │ 2024-01-15     │
└────────┴──────────┴────────────────┘
```

### 2. 自动汇率更新

**集成**：Exchange Rates API

```typescript
// 每天自动更新汇率
async function updateExchangeRates() {
  // 1. 获取公司基础货币
  const baseCurrency = company.baseCurrencyCode; // "CNY"
  
  // 2. 获取最新汇率
  const rates = await exchangeRatesAPI.getRates(baseCurrency);
  
  // 3. 更新所有启用的货币
  await updateCurrencies(rates);
}
```

### 3. 交易中使用不同货币

#### 报价单（Quote）
```typescript
{
  quoteId: "Q-001",
  customerId: "US-CUSTOMER",
  currencyCode: "USD",        // 客户货币
  exchangeRate: 7.25,         // 汇率
  items: [
    {
      description: "Product A",
      unitPrice: 100,          // USD 100
      convertedUnitPrice: 725  // CNY 725 (自动计算)
    }
  ]
}
```

#### 销售订单（Sales Order）
```typescript
{
  salesOrderId: "SO-001",
  currencyCode: "EUR",        // 客户货币
  exchangeRate: 7.83,         // EUR to CNY
  subtotal: 1000,             // EUR 1000
  convertedSubtotal: 7830     // CNY 7830 (自动计算)
}
```

#### 采购订单（Purchase Order）
```typescript
{
  purchaseOrderId: "PO-001",
  supplierId: "JP-SUPPLIER",
  currencyCode: "JPY",        // 供应商货币
  exchangeRate: 0.0485,       // JPY to CNY
  supplierUnitPrice: 10000,   // JPY 10000
  unitPrice: 485              // CNY 485 (自动计算)
}
```

## ✅ 基础货币可以修改吗？

### 答案：可以，但有限制

#### 修改路径
```
/x/settings/company → 编辑公司信息 → 修改基础货币
```

#### 限制条件

1. **没有历史交易时**
   - ✅ 可以随意修改
   - 无影响

2. **有历史交易时**
   - ⚠️ 可以修改，但会影响历史数据
   - 需要重新计算所有汇率
   - 财务报表需要重新生成

3. **最佳实践**
   - ✅ 在公司设置阶段就选对
   - ✅ 避免后期修改
   - ✅ 如果必须修改，在财务年度开始时

## 🎯 实际应用场景

### 场景 1：中国出口企业

```
基础货币：CNY（人民币）

客户：
- 美国客户 → 报价 USD，订单 USD
- 欧洲客户 → 报价 EUR，订单 EUR
- 日本客户 → 报价 JPY，订单 JPY

财务报表：
- 所有收入转换为 CNY
- 成本以 CNY 核算
- 利润以 CNY 计算
```

### 场景 2：跨国公司

```
总部：美国
基础货币：USD

子公司：
- 中国子公司 → 内部使用 CNY，报告转换为 USD
- 欧洲子公司 → 内部使用 EUR，报告转换为 USD
- 日本子公司 → 内部使用 JPY，报告转换为 USD

合并报表：所有子公司数据转换为 USD
```

### 场景 3：纯内销企业

```
基础货币：CNY

所有交易：CNY
- 客户报价：CNY
- 销售订单：CNY
- 采购订单：CNY
- 发票：CNY

优势：
- 无需汇率转换
- 财务核算简单
- 无汇兑损益
```

## 🔧 如何设置货币

### 1. Onboarding 时设置（推荐）

```
创建公司 → 选择基础货币 → 完成

建议：
- 中国企业：选择 CNY
- 美国企业：选择 USD
- 欧洲企业：选择 EUR
```

### 2. 后续启用其他货币

```
/x/settings/accounting/currencies
→ Add Currency
→ 选择货币（USD, EUR, JPY等）
→ 系统自动获取汇率
→ 保存
```

### 3. 配置自动汇率更新

```
/x/settings/integrations
→ Exchange Rates
→ 安装集成
→ 配置 API Key
→ 启用自动更新（每天）
```

## 💡 最佳实践

### 1. 选择基础货币

```
考虑因素：
✅ 公司注册地货币
✅ 主要收入来源货币
✅ 财务报告要求
✅ 税务申报货币

中国企业：
推荐：CNY
原因：
- 税务申报使用 CNY
- 银行账户主要是 CNY
- 员工工资使用 CNY
- 成本核算使用 CNY
```

### 2. 启用交易货币

```
按需启用：
✅ 只启用实际使用的货币
✅ 避免启用过多货币
✅ 定期清理不用的货币

示例：
- 出口企业：CNY + USD + EUR
- 纯内销：只用 CNY
- 跨境电商：CNY + USD + EUR + JPY
```

### 3. 汇率管理

```
自动更新：
✅ 安装 Exchange Rates 集成
✅ 每天自动更新
✅ 减少人工维护

手动更新：
✅ 重要交易前检查汇率
✅ 可以手动调整汇率
✅ 锁定历史交易汇率
```

## 🎉 总结

### 关键点

✅ **基础货币是必须的** - 用于财务核算  
✅ **可以使用多个货币** - 用于交易  
✅ **基础货币可以修改** - 但不建议  
✅ **汇率自动更新** - 通过集成  
✅ **所有交易都转换** - 最终以基础货币核算  

### 设计合理性

Carbon 的货币系统设计是**标准的 ERP 多货币架构**：

✅ **符合会计准则** - 基础货币是会计要求  
✅ **支持国际贸易** - 多货币交易  
✅ **自动汇率管理** - 减少人工维护  
✅ **灵活可扩展** - 可以随时添加货币  

### 不是断链，而是完整的多货币体系

```
基础货币（CNY）
    ↓
启用交易货币（USD, EUR, JPY）
    ↓
自动汇率更新
    ↓
交易使用任意货币
    ↓
自动转换为基础货币
    ↓
财务报表统一核算
```

这是一个**完整的、专业的多货币 ERP 系统**！🎉
