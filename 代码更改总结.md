## 更改 1: 修复 CompanySwitcher 重复显示 Bug

### 文件
`apps/erp/app/components/Layout/Topbar/CompanySwitcher.tsx`

### 问题
公司切换器弹窗中显示重复的公司条目（同一个公司显示两次）

### 根本原因
`companyGroups` 的分组逻辑有 bug：
- 当处理单公司组时，会将其移动到 "Companies" 组
- 但如果原本就有 "Companies" 组，可能导致重复添加

### 修复方案
重构分组逻辑，使用两阶段处理：
1. 第一阶段：收集需要合并的单公司组
2. 第二阶段：统一合并到 "Companies" 组

### 代码变更
```typescript
// 旧代码（有问题）
for (const [key, group] of groups) {
  if (group.companies.length === 1 && key !== "Companies") {
    const existing = result.get("Companies");
    if (existing) {
      existing.companies.push(...group.companies);
    } else {
      result.set("Companies", { ... });
    }
  } else {
    const existing = result.get(key);
    if (existing) {
      existing.companies.push(...group.companies);
    } else {
      result.set(key, group);
    }
  }
}

// 新代码（已修复）
const companiesGroup: Company[] = [];

for (const [key, group] of groups) {
  if (group.companies.length === 1 && key !== "Companies") {
    companiesGroup.push(...group.companies);
  } else {
    result.set(key, group);
  }
}

if (companiesGroup.length > 0) {
  const existing = result.get("Companies");
  if (existing) {
    existing.companies.push(...companiesGroup);
  } else {
    result.set("Companies", { name: "Companies", companies: companiesGroup });
  }
}
```

### 潜在问题
❌ **状态不持久化** - 页面刷新后状态丢失

### 改进建议
✅ 使用 localStorage 或 Zustand store 持久化状态

```typescript
  aria-label={isPinned ? "收起侧边栏" : "展开侧边栏"}
```

#### 3. 智能 hover 控制
```typescript
onMouseEnter={isPinned || editMode.isEditing ? undefined : navigationPanel.onOpen}
onMouseLeave={isPinned || editMode.isEditing ? undefined : navigationPanel.onClose}
```

### 潜在问题

#### ❌ 严重问题

1. **状态不持久化**
   - 页面刷新后 `isPinned` 状态丢失
   - 用户每次都要重新固定

2. **与现有 UI Store 不一致**
   - 项目中已有 `useUIStore` 管理侧边栏状态
   - 但 `PrimaryNavigation` 没有使用它
   - 可能导致状态不同步

3. **移动端适配问题**
   - 没有考虑移动端的行为
   - 可能在小屏幕上表现异常

#### ⚠️ 中等问题

4. **无障碍性**
   - 切换按钮的文字提示只在展开时显示
   - 收起时只有图标，可能不够清晰

5. **动画不流畅**
   - 固定/取消固定时可能有跳跃感
   - 没有过渡动画

## 完整修复代码（推荐版本）

### 1. 更新 UI Store

```typescript
// stores/ui.ts
import { create } from 'zustand';
import { persist, createJSONStorage } from 'zustand/middleware';

interface UIStore {
  isSidebarOpen: boolean;
  isSidebarPinned: boolean;
  setSidebarOpen: (isOpen: boolean) => void;
  setSidebarPinned: (isPinned: boolean) => void;
  toggleSidebar: () => void;
  toggleSidebarPin: () => void;
}

export const useUIStore = create<UIStore>()(
  persist(
    (set) => ({
      isSidebarOpen: true,
      isSidebarPinned: false,
      setSidebarOpen: (isOpen) => set({ isSidebarOpen: isOpen }),
      setSidebarPinned: (isPinned) => set({ isSidebarPinned: isPinned }),
      toggleSidebar: () => set((state) => ({ isSidebarOpen: !state.isSidebarOpen })),
      toggleSidebarPin: () => set((state) => ({ isSidebarPinned: !state.isSidebarPinned }))
    }),
    {
      name: 'carbon-ui-storage',
      storage: createJSONStorage(() => localStorage)
    }
  )
);
```

### 2. 更新 PrimaryNavigation

```typescript

    <div className="w-14 h-full flex-col z-50 hidden sm:flex">
      <nav
        data-state={isOpen ? "expanded" : "collapsed"}
        className={cn(
          "bg-background py-2 group z-10 h-full w-14 data-[state=expanded]:w-[13rem]",
          "flex flex-col justify-between data-[state=expanded]:shadow-xl data-[state=expanded]:border-r data-[state=expanded]:border-border",
```
