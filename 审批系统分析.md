# Carbon ERP 审批系统分析报告

## 概述

Carbon ERP 系统**已经实现了完整的审批系统**，这是一个统一的审批工作流框架，支持多种业务单据的审批流程。

---

## 核心功能

### 1. 支持的审批类型

系统目前支持以下单据类型的审批：

#### ✅ **采购订单审批** (Purchase Order Approval)
- 基于金额阈值的自动审批规则
- 支持多级审批（根据金额范围）
- 状态：Draft → Needs Approval → Approved/Rejected → Confirmed

#### ✅ **质量文档审批** (Quality Document Approval)
- 质量管理文档的审批流程
- 支持文档激活前的审批要求
- 状态：Draft → Pending Approval → Active/Archived

#### ✅ **供应商审批** (Supplier Approval)
- 新供应商激活前的审批流程
- 状态：Pending → Approved → Active
- 可选功能（通过配置启用）

#### ✅ **质量问题审批** (Non-Conformance Approval)
- 不合格品处理的审批要求
- 支持 MRB (Material Review Board) 审批
- 多种审批要求类型

## 数据库架构

### 核心表结构

#### 1. `approvalRequest` - 审批请求表
```sql
- id: 审批请求ID
- documentType: 单据类型 (purchaseOrder, qualityDocument)
- documentId: 关联的单据ID
- status: 审批状态 (Pending, Approved, Rejected, Cancelled)
- amount: 审批金额（用于采购订单）
- requestedBy: 申请人
- requestedAt: 申请时间
- approverId: 指定审批人
- decisionBy: 实际审批人
- decisionAt: 审批时间
- decisionNotes: 审批备注
```

#### 2. `approvalRule` - 审批规则表
```sql
- id: 规则ID
- documentType: 适用的单据类型
- enabled: 是否启用
- approverGroupIds: 审批人组ID列表
- defaultApproverId: 默认审批人
- lowerBoundAmount: 最低金额阈值
- escalationDays: 升级天数（未使用）
```

### 审批状态枚举

```typescript
type ApprovalStatus = 'Pending' | 'Approved' | 'Rejected' | 'Cancelled';
type ApprovalDocumentType = 'purchaseOrder' | 'qualityDocument';
```

## 审批规则配置

### 金额阈值规则（采购订单）

系统支持基于金额的多级审批规则：

```
规则1: $0 - $5,000    → 审批组A
规则2: $5,000 - $10,000 → 审批组B
规则3: $10,000+        → 审批组C
```

- 每个规则定义一个最低金额阈值
- 审批适用于该金额到下一个规则金额之间的订单
- 可以指定多个审批人组和默认审批人

### 质量文档规则

- 可以配置是否需要审批才能激活文档
- 不基于金额，而是基于文档类型和工作流

## 工作流程

### 采购订单审批流程

```
1. 用户创建采购订单（状态：Draft）
2. 用户点击"Request Approval"
3. 系统检查审批规则：
   - 根据订单金额匹配规则
   - 确定审批人组
4. 创建 approvalRequest 记录
5. 订单状态变更为"Needs Approval"
6. 发送通知给审批人
7. 审批人审批或拒绝：
   - Approved → 订单可以继续处理
   - Rejected → 订单返回 Draft 状态
8. 发送决策通知给申请人
```

### 质量文档审批流程

```
1. 用户创建质量文档（状态：Draft）
2. 用户尝试将状态改为"Active"
3. 系统检查是否需要审批：
   - 如果有审批规则，自动创建审批请求
   - 状态保持为 Draft，但显示"Pending Approval"
4. 发送通知给审批人
5. 审批人审批或拒绝：
   - Approved → 文档状态变为 Active
   - Rejected → 文档保持 Draft 状态
6. 发送决策通知给申请人
```

## 总结

Carbon ERP 的审批系统是一个**功能完整、设计良好**的企业级审批解决方案：

### ✅ 优点
1. **统一架构** - 单一的审批框架支持多种单据类型
2. **灵活配置** - 基于规则的审批，支持多级审批
3. **完整集成** - 与通知、权限系统深度集成
4. **用户友好** - 清晰的UI和工作流程
5. **可扩展** - 易于添加新的审批类型
