# 🚀 Carbon 服务启动说明

## ❓ 数据库需要单独启动吗？

**不需要！** `./dev.sh` 会自动启动所有服务。

---

## 📦 `./dev.sh` 自动启动的服务

运行 `./dev.sh` 时，会自动启动以下所有服务：

### 1. Docker 容器
- ✅ **PostgreSQL** - 数据库
- ✅ **Redis** - 缓存和会话存储
- ✅ **Supabase** - 后端服务（Auth、Storage、Realtime）
- ✅ **Inbucket** - 本地邮件服务器
- ✅ **Inngest** - 后台任务服务

### 2. 数据库初始化
- ✅ 运行数据库迁移（500+ 个迁移文件）
- ✅ 生成 TypeScript 类型定义
- ✅ 初始化数据库结构

### 3. 应用服务
- ✅ **ERP** - 企业资源规划系统 (端口 4000)
- ✅ **MES** - 制造执行系统 (端口 4001)

---

## 🔄 启动流程

```bash
./dev.sh
```

**自动执行的步骤**：

1. **检查依赖** - 确保 Node.js、pnpm 等已安装
2. **安装 npm 包** - 如果需要（`pnpm install`）
3. **启动 Docker 容器** - PostgreSQL、Redis、Supabase 等
4. **等待数据库就绪** - 确保 PostgreSQL 可以连接
5. **运行数据库迁移** - 创建所有表和结构
6. **生成类型定义** - 为 TypeScript 生成数据库类型
7. **启动应用** - ERP 和 MES 开发服务器
8. **显示访问地址** - 告诉你如何访问

## 💡 常见问题

### Q: 需要手动启动 Docker 吗？

**A**: 需要确保 **Docker Desktop 正在运行**，但不需要手动启动容器。`./dev.sh` 会自动启动所有需要的容器。

### Q: 需要手动运行数据库迁移吗？

**A**: 不需要。`./dev.sh` 会自动运行所有迁移。

### Q: 需要手动创建数据库吗？

**A**: 不需要。PostgreSQL 容器启动时会自动创建数据库。

### Q: 如果只想启动数据库怎么办？

**A**: 可以使用 `crbn` 命令：

```bash
# 只启动 Docker 容器（不启动应用）
crbn up --no-apps

# 停止所有服务
crbn down

# 查看状态
crbn status
```

### Q: 首次启动需要多久？

**A**: 
- **首次启动**: 5-10 分钟（需要下载 Docker 镜像、运行迁移）
- **后续启动**: 1-2 分钟（镜像已下载，只需启动容器）

## 🔧 手动控制服务

如果需要手动控制服务，可以使用 `crbn` 命令：

### 启动服务

```bash
# 启动所有服务（包括应用）
crbn up

# 只启动 Docker 容器（不启动应用）
crbn up --no-apps

# 启动并强制拉取最新镜像
crbn up --pull
```

### 停止服务

```bash
# 停止所有服务
crbn down

# 停止并删除所有数据
crbn reset
```

### 查看状态

```bash
# 查看所有服务状态
crbn status

# 查看 Docker 容器
docker ps
```
