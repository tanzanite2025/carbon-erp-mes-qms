# Carbon ERP/MES 阿里云部署方案（内部使用版）

## 部署目标

- ✅ 内部团队使用，不对外开放
- ✅ 使用公网IP直接访问，不需要域名
- ✅ 4GB内存即可运行（实测够用）
- ✅ 成本最优化
- ✅ 部署简单快速

### 安全组配置

**必须开放的端口:**
```
22   - SSH（仅限你的办公室IP，提高安全性）
80   - HTTP（所有人可访问）
3001 - ERP应用（所有人可访问）
3002 - MES应用（所有人可访问）
8000 - Supabase API（所有人可访问）
```

**不要开放的端口（内部使用）:**
```
5432 - PostgreSQL（危险！）
6379 - Redis（危险！）
3000 - Supabase Studio（可选，仅限办公室IP）
```

## 完整部署步骤

### 第一步：连接服务器

```bash
# 使用公网IP连接（从阿里云控制台获取）
ssh root@你的公网IP

# 例如：
```

### 第二步：更新系统

```bash
# 更新软件包列表

# 升级已安装的软件包

# 安装必要工具
apt install -y curl wget git nano
```


### 第五步：部署Redis

```bash
# 返回工作目录
cd /opt/carbon

# 创建Redis配置
mkdir -p redis
cd redis

# 创建docker-compose.yml
nano docker-compose.yml
```

**Redis配置内容:**
```yaml
version: '3.8'

services:
  redis:
    image: redis:7-alpine
    container_name: carbon-redis
    ports:
      - "6379:6379"
    volumes:
      - redis-data:/data
    command: redis-server --appendonly yes --requirepass 你的Redis密码
    restart: unless-stopped

volumes:
  redis-data:
    driver: local
```

**启动Redis:**
```bash
docker compose up -d

# 验证Redis
docker exec carbon-redis redis-cli -a 你的Redis密码 ping
# 应该返回: PONG
```

### 备份数据库

```bash
# 创建备份目录
mkdir -p /opt/carbon/backups

# 备份PostgreSQL
docker exec supabase-db pg_dump -U postgres postgres > /opt/carbon/backups/backup-$(date +%Y%m%d-%H%M%S).sql

# 自动备份脚本
nano /opt/carbon/backup.sh
```

**备份脚本内容:**
```bash
#!/bin/bash
BACKUP_DIR="/opt/carbon/backups"
DATE=$(date +%Y%m%d-%H%M%S)

# 备份数据库
docker exec supabase-db pg_dump -U postgres postgres > $BACKUP_DIR/db-$DATE.sql

# 压缩备份
gzip $BACKUP_DIR/db-$DATE.sql

# 删除7天前的备份
find $BACKUP_DIR -name "*.sql.gz" -mtime +7 -delete

echo "Backup completed: db-$DATE.sql.gz"
```

**设置定时备份:**
```bash
# 添加执行权限
chmod +x /opt/carbon/backup.sh

# 添加到crontab（每天凌晨2点备份）
crontab -e

# 添加以下行
0 2 * * * /opt/carbon/backup.sh >> /var/log/carbon-backup.log 2>&1
```

## 安全加固

### 1. SSH安全

```bash
# 修改SSH配置
nano /etc/ssh/sshd_config

# 修改以下配置
Port 22                          # 可以改成其他端口，如2222
PermitRootLogin yes              # 生产环境建议改为no，创建普通用户
PasswordAuthentication yes       # 建议改为no，使用密钥登录
PubkeyAuthentication yes

# 重启SSH服务
systemctl restart sshd
```

### 2. 防火墙配置

```bash
# 安装UFW防火墙
apt install -y ufw

# 允许SSH（重要！先设置这个，否则会断开连接）
ufw allow 22/tcp

# 允许应用端口
ufw allow 80/tcp
ufw allow 3001/tcp
ufw allow 3002/tcp
ufw allow 8000/tcp

# 启用防火墙
ufw enable

# 查看状态
ufw status
```

### 3. 安装Fail2ban（防止暴力破解）

```bash
# 安装Fail2ban
apt install -y fail2ban

# 启动服务
systemctl start fail2ban
systemctl enable fail2ban

# 查看状态
fail2ban-client status
```

### 4. 定期更新系统

```bash
# 创建更新脚本
nano /opt/carbon/update-system.sh
```

**更新脚本内容:**
```bash
#!/bin/bash
apt update
apt upgrade -y
apt autoremove -y
apt autoclean
```

```bash
# 添加执行权限
chmod +x /opt/carbon/update-system.sh

# 每周日凌晨3点自动更新
crontab -e
# 添加：
0 3 * * 0 /opt/carbon/update-system.sh >> /var/log/system-update.log 2>&1
```

## 常见问题

### Q: 为什么不用域名？
A: 内部使用不需要域名，直接用IP访问更简单，省钱省事。员工保存书签即可。

### Q: 没有HTTPS安全吗？
A: 内部网络使用HTTP问题不大。如果需要更高安全性，可以配置VPN或内网穿透。

### Q: 4GB内存真的够用吗？
A: 够用。实测PostgreSQL占用500MB-1GB，Redis占用100-200MB，应用占用1-2GB，总共2-3GB，留有余量。

### Q: 如何添加新用户？
A: 登录后在"设置 → 用户管理"中添加，或使用超级管理员账号创建。

### Q: 数据会丢失吗？
A: 不会。PostgreSQL和Redis都配置了数据持久化，重启不会丢失数据。建议定期备份。

### Q: 可以从外网访问吗？
A: 可以。只要有公网IP，任何地方都能访问。如果只想内网访问，配置安全组限制IP即可。

### Q: 如何限制只有公司IP能访问？
A: 在阿里云安全组中，将端口3001、3002、8000的访问来源改为"授权对象：你的公司公网IP"。
