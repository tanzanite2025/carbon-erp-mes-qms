# 快速参考：生产部署首次登录

## 核心概念

**Carbon 没有内置超级管理员账号！**

第一个注册的用户自动成为公司 Owner（拥有所有权限）。

## 三种登录方式

### 1. 邮箱登录（最简单）

```bash
# 前提：必须配置邮件服务
1. 访问 https://your-domain.com/login
2. 输入邮箱地址
3. 接收验证码或 Magic Link
4. 完成 Onboarding（创建公司）
```

**优点：** 简单，无需额外配置  
**缺点：** 需要配置邮件服务

### 2. OAuth 登录（推荐生产环境）

```bash
# Google OAuth
AUTH_PROVIDERS=google
GOOGLE_CLIENT_ID=your-client-id
GOOGLE_CLIENT_SECRET=your-client-secret

# Azure OAuth（适合企业）
AUTH_PROVIDERS=azure
AZURE_CLIENT_ID=your-client-id
AZURE_CLIENT_SECRET=your-client-secret
AZURE_TENANT_ID=your-tenant-id
```

**优点：** 安全，无需配置邮件  
**缺点：** 需要申请 OAuth 应用

### 3. 开发绕过（仅开发环境）

```bash
# .env
DEV_BYPASS_EMAIL=admin@example.com
```

**⚠️ 生产环境必须删除此配置！**

## 最小化生产配置

### 必需配置

```bash
# .env

# Supabase（数据库和认证）
SUPABASE_URL=https://your-project.supabase.co
SUPABASE_ANON_KEY=your-anon-key
SUPABASE_SERVICE_ROLE=your-service-role-key

# 应用 URL
ERP_URL=https://your-domain.com
MES_URL=https://mes.your-domain.com

# 版本
CARBON_EDITION=enterprise  # 或 cloud, community

# 语言
DEFAULT_LANGUAGE=zh
```

### 邮件配置（二选一）

#### 选项 A：使用 Supabase 内置邮件

无需额外配置，但有发送限制。

#### 选项 B：配置 SMTP（推荐）

在 Supabase Dashboard 配置：
- Authentication > Email Templates > SMTP Settings

```
SMTP Host: smtp.yourprovider.com
SMTP Port: 587
SMTP User: your-email@yourcompany.com
SMTP Password: your-password
```

## 首次登录流程

```
1. 访问登录页面
   ↓
2. 输入邮箱或使用 OAuth
   ↓
3. 验证身份（邮件验证码或 OAuth）
   ↓
4. Onboarding 流程：
   - 设置个人信息
   - 选择界面主题
   - 创建公司信息
   - 选择订阅计划（Cloud 版本）
   ↓
5. 成为公司 Owner
```

## 添加其他用户

### 方法：通过邀请

```
1. 进入 Users > Employees
2. 点击 "New Employee"
3. 填写员工信息（姓名、邮箱、职位）
4. 系统发送邀请邮件
5. 员工点击邮件链接加入
```

## 三种版本的区别

| 版本 | 用户注册 | 适用场景 |
|------|---------|---------|
| **Community** | 任何人都可以注册 | 开源社区 |
| **Cloud** | 任何人都可以注册 + 付费 | SaaS 服务 |
| **Enterprise** | 仅邀请制 | 私有部署 |

## 常见问题

### Q: 忘记密码怎么办？

A: Carbon 使用 Magic Link，没有密码。重新输入邮箱即可收到新的登录链接。

### Q: 如何禁用自助注册？

A: 设置 `CARBON_EDITION=enterprise`

### Q: 可以使用手机号登录吗？

A: 可以，但需要额外配置。详见 `中国本地化-手机号登录方案.md`

### Q: 第一个用户有什么权限？

A: 第一个用户自动成为 Owner，拥有所有权限，包括：
- 管理所有用户
- 配置系统设置
- 删除公司
- 分配权限

## 安全检查清单

- [ ] 配置数据库备份
