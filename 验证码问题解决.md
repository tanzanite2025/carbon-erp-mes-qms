## 🔍 修改的代码

### ERP (`apps/erp/app/routes/_public+/login.tsx`)

```typescript
const devBypassEmail = process.env.DEV_BYPASS_EMAIL;
if (
  devBypassEmail &&
  email.toLowerCase() === devBypassEmail.toLowerCase()
) {
  // 如果用户不存在，自动创建
  if (!user.data) {
    const { createUser } = await import("@carbon/auth/users.server");
    await createUser(email);
  }
  
  // 直接登录
  const authSession = await signInWithBypassEmail(email);
  if (authSession) {
    const sessionCookie = await setAuthSession(request, { authSession });
    return redirect(path.to.authenticatedRoot, {
      headers: [["Set-Cookie", sessionCookie]]
    });
  }
}
```

### MES (`apps/mes/app/routes/_public+/login.tsx`)

同样的修改。

## 💡 工作原理

### 之前的问题

1. 输入 `admin@example.com`
2. 系统检查用户是否存在
3. **用户不存在** → 走注册流程 → 需要验证码 ❌

### 现在的解决方案

1. 输入 `admin@example.com`
2. 系统检查是否是开发绕过邮箱
3. **是开发绕过邮箱** → 自动创建用户（如果不存在）→ 直接登录 ✅

---

## 📝 注意事项

- ✅ 只对 `admin@example.com` 有效（`.env` 中配置的邮箱）
- ✅ 仅用于开发环境
- ✅ 生产环境不应该设置 `DEV_BYPASS_EMAIL`
- ✅ 自动创建的用户会被激活
