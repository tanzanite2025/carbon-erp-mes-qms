# 快速参考：超级管理员登录

## 配置（2 步）

### 1. 编辑 .env

```bash
# 内置超级管理员账号
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="YourSecurePassword123!"
```

### 2. 重启服务

```bash
crbn dev
```

## 登录（3 步）

1. 访问 `http://localhost:4000/login`
2. 输入邮箱：`admin@yourcompany.com`
3. 输入密码：`YourSecurePassword123!`

**首次登录会自动创建账号并进入 Onboarding 流程。**

## 开发 vs 生产

### 开发环境

```bash
# 两种方式都可以用
DEV_BYPASS_EMAIL="admin@example.com"        # 无密码，自动登录
SUPER_ADMIN_EMAIL="admin@dev.local"         # 有密码，更安全
SUPER_ADMIN_PASSWORD="admin123"
```

### 生产环境

```bash
# ⚠️ 必须删除 DEV_BYPASS_EMAIL
# DEV_BYPASS_EMAIL="admin@example.com"

# 只使用超级管理员
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="Xy9$mK2#pL8@vN4&qR7!"  # 强密码！
```

## 密码要求

生产环境密码必须：
- ✅ 至少 12 位
- ✅ 包含大小写字母
- ✅ 包含数字
- ✅ 包含特殊字符

## 常见问题

### 忘记密码？

修改 `.env` 中的 `SUPER_ADMIN_PASSWORD`，然后重启服务。

### 可以有多个超级管理员吗？

不可以。但可以登录后添加其他管理员：
```
Users > Employees > New Employee > 设置角色为 Admin
```

### 密码输入框不显示？

确保输入的邮箱与 `SUPER_ADMIN_EMAIL` 完全一致（包括大小写）。

## 安全提示

- ⚠️ 生产环境必须使用强密码（20+ 位）
- ⚠️ 生产环境必须删除 `DEV_BYPASS_EMAIL`
- ⚠️ 定期更换密码（3-6 个月）
- ⚠️ 监控登录日志
