# 内置超级管理员账号 - 使用指南

## 功能说明

为了方便生产环境部署后直接登录，我已经为你添加了**内置超级管理员账号**功能。

### 特点

- ✅ **开发和生产环境都可用** - 不像 `DEV_BYPASS_EMAIL` 只能用于开发
- ✅ **使用邮箱+密码登录** - 安全可靠
- ✅ **首次登录自动创建** - 无需手动创建用户
- ✅ **跨环境一致** - 开发、测试、生产使用同一套账号
- ✅ **支持 ERP 和 MES** - 两个应用都可以使用

## 配置方法

### 1. 设置环境变量

在 `.env` 文件中添加：

```bash
# 内置超级管理员账号
SUPER_ADMIN_EMAIL="superadmin@yourcompany.com"
SUPER_ADMIN_PASSWORD="YourSecurePassword123!"
```

**重要提示：**
- 邮箱可以是任意有效的邮箱地址
- 密码必须足够安全（建议至少 12 位，包含大小写字母和数字）
- 生产环境必须使用强密码！

### 2. 重启服务

```bash
# 开发环境
crbn dev

# 或者生产环境
docker-compose restart
```

## 使用方法

### 登录步骤

1. **访问登录页面**
   ```
   开发环境：http://localhost:4000/login
   生产环境：https://your-domain.com/login
   ```

2. **输入超级管理员邮箱**
   - 输入你在 `.env` 中配置的 `SUPER_ADMIN_EMAIL`
   - 例如：`superadmin@yourcompany.com`

3. **自动显示密码输入框**
   - 当你输入超级管理员邮箱后，密码输入框会自动出现
   - 这是为了区分普通用户（使用 Magic Link）和超级管理员（使用密码）

4. **输入密码**
   - 输入你在 `.env` 中配置的 `SUPER_ADMIN_PASSWORD`

5. **点击登录**
   - 首次登录会自动创建超级管理员账号
   - 后续登录直接验证密码

6. **完成 Onboarding**
   - 首次登录需要完成 Onboarding 流程
   - 设置个人信息、主题、公司信息等

## 工作原理

### 前端逻辑

```typescript
// 检测是否是超级管理员邮箱
const superAdminEmail = window.env?.SUPER_ADMIN_EMAIL;
const isSuperAdmin = emailInput === superAdminEmail;

// 如果是超级管理员，显示密码输入框
{showPassword && (
  <Input
    name="password"
    type="password"
    placeholder="Password"
  />
)}
```

### 后端逻辑

```typescript
// 1. 检查是否是超级管理员邮箱
if (email === SUPER_ADMIN_EMAIL) {
  // 2. 验证密码
  if (password !== SUPER_ADMIN_PASSWORD) {
    return error("Invalid credentials");
  }
  
  // 3. 如果用户不存在，自动创建
  if (!user) {
    await carbonClient.auth.signUp({
      email: SUPER_ADMIN_EMAIL,
      password: SUPER_ADMIN_PASSWORD,
      options: {
        data: {
          firstName: "Super",
          lastName: "Admin"
        }
      }
    });
  }
  
  // 4. 使用密码登录
  const { data } = await carbonClient.auth.signInWithPassword({
    email: SUPER_ADMIN_EMAIL,
    password: SUPER_ADMIN_PASSWORD
  });
  
  // 5. 设置 Session 并重定向
  return redirect("/x");
}
```

## 安全建议

### 开发环境

```bash
# 开发环境可以使用简单密码
SUPER_ADMIN_EMAIL="admin@dev.local"
SUPER_ADMIN_PASSWORD="admin123"
```

### 生产环境

```bash
# 生产环境必须使用强密码
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="Xy9$mK2#pL8@vN4&qR7!"

# 建议使用密码生成器生成 20+ 位随机密码
```

### 密码要求

- ✅ 至少 12 位字符
- ✅ 包含大写字母
- ✅ 包含小写字母
- ✅ 包含数字
- ✅ 包含特殊字符
- ✅ 不使用常见密码（如 password123）
- ✅ 不使用公司名称或个人信息

### 额外安全措施

1. **限制 IP 访问**（可选）
   ```nginx
   # Nginx 配置
   location /login {
       allow 192.168.1.0/24;  # 只允许内网访问
       deny all;
   }
   ```

2. **启用 2FA**（未来功能）
   - 可以考虑为超级管理员启用两步验证

3. **定期更换密码**
   - 建议每 3-6 个月更换一次密码

4. **监控登录日志**
   - 检查 Supabase Auth 日志
   - 监控异常登录尝试

## 与其他登录方式的区别

| 登录方式 | 适用环境 | 需要邮件服务 | 安全性 | 使用场景 |
|---------|---------|------------|-------|---------|
| **超级管理员** | 开发+生产 | ❌ 不需要 | ⭐⭐⭐⭐ | 首次部署、紧急访问 |
| **DEV_BYPASS_EMAIL** | 仅开发 | ❌ 不需要 | ⭐⭐ | 本地开发测试 |
| **Magic Link** | 开发+生产 | ✅ 需要 | ⭐⭐⭐⭐⭐ | 普通用户登录 |
| **OAuth (Google/Azure)** | 开发+生产 | ❌ 不需要 | ⭐⭐⭐⭐⭐ | 企业用户登录 |

## 常见问题

### Q1: 忘记超级管理员密码怎么办？

**方法 1：修改 .env 文件**
```bash
# 修改密码
SUPER_ADMIN_PASSWORD="NewPassword123!"

# 重启服务
crbn dev  # 或 docker-compose restart
```

**方法 2：通过 Supabase Dashboard 重置**
1. 登录 Supabase Dashboard
2. 进入 Authentication > Users
3. 找到超级管理员用户
4. 点击 "Reset Password"

### Q2: 可以有多个超级管理员吗？

目前只支持一个内置超级管理员。如果需要多个管理员：

1. 使用超级管理员登录
2. 进入 Users > Employees
3. 添加其他管理员
4. 设置角色为 "Admin" 或 "Owner"

### Q3: 超级管理员和普通管理员有什么区别？

**超级管理员：**
- 使用邮箱+密码登录
- 无需邮件验证
- 首次登录自动创建
- 适合首次部署和紧急访问

**普通管理员：**
- 使用 Magic Link 或 OAuth 登录
- 需要邮件验证或 OAuth 授权
- 通过邀请加入
- 适合日常使用

权限上两者相同，都是 Owner 角色。

### Q4: 生产环境需要删除 DEV_BYPASS_EMAIL 吗？

**是的！** 生产环境必须删除或注释掉 `DEV_BYPASS_EMAIL`：

```bash
# 开发环境
DEV_BYPASS_EMAIL="admin@example.com"  # 保留

# 生产环境
# DEV_BYPASS_EMAIL="admin@example.com"  # 必须注释掉！

# 两种环境都可以使用
SUPER_ADMIN_EMAIL="admin@yourcompany.com"
SUPER_ADMIN_PASSWORD="SecurePassword123!"
```

### Q5: 如何在生产环境隐藏密码输入框？

密码输入框只在输入超级管理员邮箱时显示，普通用户看不到。

如果你想完全隐藏超级管理员功能：
```bash
# 不设置这两个环境变量即可
# SUPER_ADMIN_EMAIL=""
# SUPER_ADMIN_PASSWORD=""
```

### Q6: 超级管理员账号会被锁定吗？

不会。超级管理员账号不受 Rate Limit 限制（目前实现）。

但建议添加 Rate Limit 保护：
```bash
# .env
RATE_LIMIT="100"  # 限制每小时登录尝试次数
```
